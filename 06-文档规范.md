## 二、代码注释文档

### 2.1 函数/方法注释

```python
def create_order(
    user_id: str,
    product_id: str,
    quantity: int
) -> Order:
    """创建订单。

    Args:
        user_id: 用户 ID
        product_id: 商品 ID
        quantity: 购买数量，必须大于 0

    Returns:
        创建的订单对象

    Raises:
        ValueError: 库存不足时抛出
        UserNotFoundError: 用户不存在时抛出
    """
```

### 2.2 类注释
```python
class OrderService:
    """订单服务，负责订单的创建、查询、取消。

    使用示例：
        service = OrderService(db)
        order = await service.create(user_id, product_id, 2)
    """
```

## 三、接口文档

### 3.1 接口描述模板
```markdown
## POST /api/orders

创建新订单

### 请求头
Authorization: Bearer <token>

### 请求体
{
    "product_id": "prod_123",
    "quantity": 2,
    "remark": "加急"  // 可选
}

### 成功响应 (201)
{
    "code": 0,
    "data": {
        "order_id": "ord_456",
        "status": "pending",
        "total_amount": 199.00
    }
}

### 错误码
- 40001: 参数错误
- 40002: 商品不存在
- 40003: 库存不足
```

## 四、设计文档

### 4.1 何时写
- 需求不清晰时，先写文档理清思路
- 复杂功能（涉及 3+ 模块）
- 重大架构变更

### 4.2 设计文档模板
```markdown
# 功能设计：xxx

## 背景
为什么做

## 目标
完成后的效果

## 方案
具体怎么做（附架构图/流程图）

## 影响范围
涉及哪些模块

## 风险
可能的问题
```

## 五、文件命名

| 类型 | 格式 | 示例 |
|------|------|------|
| 普通文档 | 中文或英文描述 | `架构设计.md` |
| 日期相关 | YYYY-MM-DD 前缀 | `2026-05-20-会议纪要.md` |
| 会议记录 | 日期 + 主题 | `2026-05-20-技术评审.md` |
| 版本相关 | v + 版本号 | `v2.0-更新说明.md` |

### 命名原则
- 禁止空格，使用 `-` 连接
- 禁止特殊字符
- 同名文件用前缀区分

## 六、文档维护

### 6.1 更新时机
- 功能变更时，同步更新对应文档
- 发现文档过时时，立即修复
- 每季度检查一次文档有效性

### 6.2 废弃处理
- 过时文档移动到 `archive/` 目录
- 在文件头部标注废弃日期
- 不直接删除（保留历史参考）

### 6.3 工具推荐
- 格式：Markdown（.md）
- 图表：Mermaid（代码生成流程图）
- 协作：语雀 / Notion / Confluence
