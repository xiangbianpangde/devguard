# ADR 0001: 规范体系结构

> 日期: 2026-05-26
> 状态: 已采纳

## 背景

需要为「通用开发规范」项目确定核心体系结构：覆盖哪些维度、使用什么格式、如何组织文件。

## 决策

### 六维度体系

选定架构、代码、Git、API、测试、文档六个维度。

**理由**：
- 覆盖软件开发生命周期从设计到交付的完整链路
- 每个维度独立可读，无强耦合
- 六个文档数量适中，不冗长不遗漏

### 纯 Markdown 格式

所有规范文档使用 Markdown。

**理由**：
- Git 友好：diff 可读，merge 无冲突
- 可移植：任何编辑器、任何平台
- AI 兼容：LLM 天然擅长处理 Markdown
- 可转换：通过 Pandoc 等工具可输出 HTML/PDF/DOCX

### 文件命名：序号 + 中文描述

使用 `01-架构设计规范.md` 而非 `architecture-standards.md`。

**理由**：
- 中文读者直接理解文件内容
- 序号保证阅读顺序
- 兼容中英文混合环境

## 后果

- 制定规范时需注意 Markdown 的表达力限制（复杂表格、图表需借助 ASCII art 或外部工具）
- 未来如需国际化，需额外维护英文版本
