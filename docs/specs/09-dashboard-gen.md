# BDD 规格: 仪表盘自动生成脚本

> 对应 `dashboard.html` 的数据生成
> 更新: 2026-05-26

---

## 功能：仪表盘自动生成脚本

### 场景 1：从 STATUS.md 解析项目进度

- 前置条件：`STATUS.md` 存在且格式正确
- 操作步骤：
  1. 运行生成脚本
  2. 打开 `dashboard.html`
- 预期结果：
  - 进度条显示已完成/总功能点的百分比
  - 每个功能点以卡片形式展示（名称、BDD 引用、状态、完成日期）
  - 阻塞项独立区域展示
  - 技术债独立区域展示

### 场景 2：无需手动编辑 dashboard.html

- 前置条件：`STATUS.md` 更新后（如标记新功能点为已完成）
- 操作步骤：
  1. 修改 `STATUS.md` 中某功能点状态
  2. 刷新浏览器中的 `dashboard.html`
- 预期结果：
  - dashboard 自动反映最新状态
  - 无需手动修改 HTML 代码

### 场景 3：离线可用（纯静态）

- 前置条件：无网络连接
- 操作步骤：
  1. 通过 HTTP 服务打开 `dashboard.html`
- 预期结果：
  - 不依赖外部 CDN、API 或数据库
  - 所有样式和逻辑内联在单个 HTML 文件中
  - 仅通过 `fetch('STATUS.md')` 获取数据

### 场景 4：状态更新无需重启服务

- 前置条件：服务器正在运行
- 操作步骤：
  1. 修改 `STATUS.md`
  2. 等待自动刷新（60 秒）或手动刷新浏览器
- 预期结果：
  - 新状态立即反映在仪表盘上
  - 无需重启 HTTP 服务
