# BDD 规格: 架构设计规范

> 对应 `01-架构设计规范.md`

---

## 功能：架构设计规范文档

### 场景 1：核心原则完整覆盖
- 前置条件：无
- 操作步骤：
  1. 打开 `01-架构设计规范.md`
  2. 逐条阅读"核心原则"章节
- 预期结果：
  - 至少包含单一职责、开闭原则、依赖倒置、接口隔离 4 条原则
  - 每条原则包含**描述**和**为什么**两段
  - 每条原则的文字不超过 5 行

### 场景 2：分层架构清晰可执行
- 前置条件：读者已理解核心原则
- 操作步骤：
  1. 阅读"分层架构"章节
  2. 对照 ASCII 图验证依赖方向
- 预期结果：
  - 包含表现层/应用层/领域层/基础设施层四层定义
  - 每层有明确的职责和不包含项
  - 依赖方向明确：上层→下层，下层不反向依赖

### 场景 3：模块划分有选择标准
- 前置条件：读者面临新项目搭建
- 操作步骤：
  1. 阅读"模块划分"章节
  2. 根据项目规模选择划分方式
- 预期结果：
  - 提供至少两种划分方案（按业务领域 / 按技术层次）
  - 包含选择建议：什么情况选哪个
  - 有示例目录结构

### 场景 4：反模式与正例对比
- 前置条件：读者已完成代码编写，需要自查
- 操作步骤：
  1. 阅读"反模式"章节
  2. 逐条对比自己的代码
- 预期结果：
  - 每种反模式有错误代码 + 正确代码对照
  - 有检查清单方便自查

### 场景 5：检查清单可操作
- 前置条件：Code Review 或提交前
- 操作步骤：
  1. 逐项核对检查清单
- 预期结果：
  - 清单用 checkbox 格式
  - 每项可以用"是/否"回答
  - 不少于 5 项
