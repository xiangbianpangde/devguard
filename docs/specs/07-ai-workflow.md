# BDD 规格: AI 协作开发流程

> 对应 `AI协作开发流程.md`

---

## 功能：AI 协作开发流程文档

### 场景 1：角色分工无歧义
- 前置条件：新成员加入项目，需要了解 AI 角色
- 操作步骤：
  1. 阅读"角色分工"章节
- 预期结果：
  - 表格包含 Claude（执行者）、DeepSeek（审查者）、Opus（架构师+开发）、"我"（决策者）四个角色
  - 每个角色有明确的职责范围，不重叠不遗漏

### 场景 2：从零开始的项目可按流程启动
- 前置条件：项目只有一份 README 或完全为空
- 操作步骤：
  1. 按双轨制（长程/思考）+ 端到端流程执行
- 预期结果：
  - 思考设计：生成 `docs/research/` 下的调研文档
  - 第一步：生成 `docs/plan/背景.md`、`docs/plan/开发清单.md`、BDD 规格、STATUS.md
  - 第二步：按功能点循环迭代开发

### 场景 3：基础设施创建命令可直接复制执行
- 前置条件：仓库未初始化
- 操作步骤：
  1. 复制 1.9 节中的 bash 命令
  2. 在终端中粘贴执行
- 预期结果：
  - git 仓库初始化成功
  - 所有目录和骨架文件创建成功
  - CLAUDE.md 内容完整

### 场景 4：开发循环不跳过关键步骤
- 前置条件：完成一个功能点的代码编写
- 操作步骤：
  1. 按第二步的 7 步循环逐一执行
- 预期结果：
  - 每个步骤都有明确入口（谁做、做什么）
  - 可观测验证不跳过（留下他人可复核的客观证据，形式由功能类型决定）
  - Git commit 格式符合 Conventional Commits

### 场景 5：MCP 代码理解工具集成
- 前置条件：开发循环中需要探索代码库
- 操作步骤：
  1. 阅读"工具链增强"章节
  2. 在仓库扫描阶段配置 CodeGraph
- 预期结果：
  - 明确 4 种推荐工具（CodeGraph / CodeGraphContext / Understand-Anything / LLM Wiki）
  - 明确图查询替代 grep 的理由（快 100-1000 倍）
  - 明确三阶段集成方案（扫描 → 实现 → 审查）
  - 明确 Token 节省效果（基准 57%）

### 场景 6：异常 — MCP 工具不可用时的 fallback 策略
- 前置条件：CodeGraph 或 MCP server 宕机/网络不通
- 操作步骤：
  1. 开发循环的仓库扫描阶段
  2. 图查询工具返回连接超时
- 预期结果：
  - 自动回退到传统 grep/read_file 模式（不影响开发流程）
  - 工具状态写入 CLAUDE.md 顶部标注（如 `⚠️ MCP 工具不可用，本次使用 grep 回退`）
  - 恢复后自动切换回图查询模式
  - 规范明确：工具不可用不应成为阻塞开发的理由
