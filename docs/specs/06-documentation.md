# BDD 规格: 文档规范

> 对应 `06-文档规范.md`

---

## 功能：文档规范文档

### 场景 1：README 模板可复制使用
- 前置条件：新项目需要创建 README
- 操作步骤：
  1. 阅读"README 规范"章节
  2. 按模板填写
- 预期结果：
  - 提供 README 模板（项目名/简介/快速开始/文档/贡献/许可）
  - 30 秒内能理解项目做什么
  - 有真实项目 README 作为正例

### 场景 2：API 文档自动生成优先
- 前置条件：API 开发完成
- 操作步骤：
  1. 阅读"API 文档规范"章节
- 预期结果：
  - 推荐 OpenAPI 3.x 作为文档标准
  - 明确手写文档仅在自动生成不可用时作为备选
  - 有 OpenAPI 示例片段

### 场景 3：变更日志格式统一
- 前置条件：新版本发布
- 操作步骤：
  1. 阅读"变更日志"章节
  2. 按 Keep a Changelog 格式更新
- 预期结果：
  - 明确 Added/Changed/Deprecated/Removed/Fixed/Security 分类
  - 有格式示例
  - 明确更新时机（每次发布）

### 场景 4：文档腐烂检查
- 前置条件：项目运行 3 个月以上
- 操作步骤：
  1. 阅读"文档维护"章节
- 预期结果：
  - 有定期检查机制描述（每月/每版本）
  - 有过时文档的处理策略（归档/更新/删除）
  - 有文档与代码同步的原则
