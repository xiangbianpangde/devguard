# BDD 规格: 文档规范

> 对应 `06-documentation_文档规范.md`（v2.0：§2.5 更新记录 + 07-汇报 的细化）
> 更新: 2026-07-11

---

## 功能：文档规范文档

### 场景 1：声明所细化的流程环节
- 前置条件：无
- 操作步骤：
  1. 打开 `06-documentation_文档规范.md`
  2. 阅读开头引言
- 预期结果：
  - 明确声明细化「§2.5 更新记录（worklog/STATUS/CLAUDE）」与「07-汇报」及交付物文档
  - 声明模板权威在 docs/templates/，本规范不重抄模板正文
  - 含档位裁剪

### 场景 2：红线配验证手段
- 前置条件：审查前
- 操作步骤：
  1. 阅读 §一「红线」
- 预期结果：
  - 红线含：必有 README（快速开始）、发布更新 CHANGELOG、文档随码同 PR、公共 API 有 docstring、注释与代码一致
  - 每条标注验证手段

### 场景 3：模板复用而非重抄
- 前置条件：新项目写文档
- 操作步骤：
  1. 阅读 §二「落地」
- 预期结果：
  - 「要写的文档 → 用哪个模板」映射表，链接到 docs/templates/
  - docstring 范例
  - CHANGELOG 六分类，强调用户视角

### 场景 4：决策表覆盖设计文档 / 命名 / 知识互联 / 收束维护
- 前置条件：文档体系建立
- 操作步骤：
  1. 阅读 §三决策表
- 预期结果：
  - 何时写设计文档（决策表）+ 五段结构
  - 文件命名表（kebab-case）
  - 知识互联：文档间双向链接（设计 ↔ BDD ↔ 规范）—— 此项保留在 06
  - 收束节点文档一致性检查 + 过时文档归档

### 场景 5：反模式与正例对比
- 前置条件：自查文档
- 操作步骤：
  1. 阅读 §四反模式
- 预期结果：
  - 含「README 有名无实」「CHANGELOG 复制 git log」「注释与代码不符」三类
  - 每类有反例 + 正例

### 场景 6：代码地图 CODE_MAP 内容已移交 08
- 前置条件：读者寻找 CODE_MAP.md 模板与要求
- 操作步骤：
  1. 在 06 中查找代码地图
- 预期结果：
  - 06 保留「双向链接 / 知识互联」要求
  - CODE_MAP 的模板与构建移交 08，§三 / §六以一句话指向 08
