# BDD 规格: 代码编写规范

> 对应 `02-代码编写规范.md`

---

## 功能：代码编写规范文档

### 场景 1：命名规范统一可执行
- 前置条件：团队需要统一的命名约定
- 操作步骤：
  1. 打开 `02-代码编写规范.md`
  2. 阅读"命名规范"章节
- 预期结果：
  - 覆盖变量/函数/类/文件/常量的命名规则
  - 每种命名规则有正确示例和错误示例
  - 明确使用 camelCase / PascalCase / snake_case / kebab-case 的场景

### 场景 2：注释规范区分必要与冗余
- 前置条件：开发者不确定何时该写注释
- 操作步骤：
  1. 阅读"注释规范"章节
- 预期结果：
  - 明确什么必须注释（公共API、复杂算法、魔法数字）
  - 明确什么不应注释（自解释代码）
  - 有注释格式示例

### 场景 3：错误处理有统一模式
- 前置条件：项目需要统一的错误处理方式
- 操作步骤：
  1. 阅读"错误处理"章节
- 预期结果：
  - 定义错误类型的分层（业务异常/系统异常/参数异常）
  - 有统一错误码规范
  - 有错误日志记录标准

### 场景 4：安全检查清单可执行
- 前置条件：提交代码前
- 操作步骤：
  1. 阅读"安全规范"章节
  2. 逐项对照代码检查
- 预期结果：
  - 覆盖 SQL 注入/XSS/CSRF/敏感信息泄露
  - 有正确做法示例
  - 不少于 5 条检查项
