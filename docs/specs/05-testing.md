# BDD 规格: 测试规范

> 对应 `05-测试规范.md`

---

## 功能：测试规范文档

### 场景 1：测试分层比例有指导
- 前置条件：项目需要制定测试策略
- 操作步骤：
  1. 阅读"测试金字塔"章节
- 预期结果：
  - 明确单元测试/集成测试/E2E 的定义和边界
  - 给出推荐比例（70%/20%/10%）
  - 有 ASCII 金字塔图示

### 场景 2：单元测试编写有模板
- 前置条件：开发者编写第一个单元测试
- 操作步骤：
  1. 阅读"单元测试"章节
  2. 按 AAA 模式（Arrange/Act/Assert）编写
- 预期结果：
  - 有 AAA 结构说明
  - 有命名规范（test_方法_场景_期望）
  - 有完整代码示例

### 场景 3：Mock 策略不滥用
- 前置条件：测试需要隔离外部依赖
- 操作步骤：
  1. 阅读"Mock 策略"章节
- 预期结果：
  - 明确什么该 Mock（数据库/外部API/时间）
  - 明确什么不该 Mock（值对象/纯函数）
  - 有 Mock 代码示例

### 场景 4：覆盖率有底线
- 前置条件：CI 流水线运行测试
- 操作步骤：
  1. 阅读"覆盖率"章节
- 预期结果：
  - 明确行覆盖率底线（建议 ≥ 80%）
  - 明确分支覆盖率建议（建议 ≥ 70%）
  - 明确哪些代码可以排除（配置/常量/自动生成）
