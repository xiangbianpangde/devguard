## 功能：测试规范文档

### 场景 1：声明所细化的流程环节
- 前置条件：无
- 操作步骤：
  1. 打开 `05-testing_测试规范.md`
  2. 阅读开头引言
- 预期结果：
  - 明确声明细化「§2.1 TDD 自检」与「§2.2 可观测验证」
  - 说明测试全绿是重构/算法/配置类功能点的合格证据
  - 含档位裁剪（轻量不强制覆盖率门禁）

### 场景 2：红线配检测手段
- 前置条件：审查前
- 操作步骤：
  1. 阅读 §一「红线」
- 预期结果：
  - 红线含：测试独立不依赖顺序、只 Mock 外部边界、覆盖正常+边界+异常、无 flaky test
  - 每条标注检测/验证手段

### 场景 3：TDD 节奏 + 覆盖率门禁可复制
- 前置条件：新项目落地测试
- 操作步骤：
  1. 阅读 §二「落地」
  2. 复制配置
- 预期结果：
  - 含 TDD 红-绿-重构循环说明
  - 有 AAA 模板 + 命名规范 `test_方法_场景_期望` + 代码示例
  - 提供可复制的 `pyproject.toml`（`--cov-fail-under` 门禁）+ CI 片段

### 场景 4：金字塔 / 该测什么 / 覆盖率 / Mock 以决策表给出
- 前置条件：制定测试策略
- 操作步骤：
  1. 阅读 §三决策表
- 预期结果：
  - 金字塔比例（≈70/20/10）
  - 覆盖率目标按档位/阶段分级（原型不要求 → 核心 ≥90%）
  - Mock 边界表（该 / 不该）

### 场景 5：反模式与正例对比
- 前置条件：自查测试
- 操作步骤：
  1. 阅读 §四反模式
- 预期结果：
  - 含「依赖执行顺序」「Mock 错对象」「只测 happy path」三类
  - 每类有反例 + 正例
