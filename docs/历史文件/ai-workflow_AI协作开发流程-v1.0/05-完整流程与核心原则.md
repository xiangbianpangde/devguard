# AI 协作开发流程 — 完整流程与核心原则

> 更新: 2026-07-11

## 完整流程总览

```
零、调研
  └─ 产出 → docs/research/

一、计划
  1. 流程笔记 (docs/plan/背景.md)
  2. 功能拆分 (STATUS.md)
  3. BDD 规格 (docs/specs/)
  4. 技术选型 (STATUS.md 中标注)
  5. DeepSeek 讨论完善（概念卡壳对话 5 轮以上仍未理解，才为该问题建专门笔记文件）
  6. DeepSeek 生成计划提纲 (STATUS.md)
  7. Opus 交叉审查 (附仓库状态)
  8. 创建基础设施 (git/STATUS/日志/仪表盘)

二、开发（功能点循环）                              ← 每个功能点完成 → 功能点汇报（对话内联，不落盘）
  ┌─────────────────────────────────┐
  │ 0. 扫描仓库 → 更新 CLAUDE.md     │
  │ 1. Opus 实现                    │
  │ 2. 可观测验证（必须）               │
  │ 3. DeepSeek 审查 diff              │
  │ 4. 你确认差异点                     │
  │ 5. 更新 worklog + STATUS               │
  │ 6. Git commit + tag               │
  │ 7. 3 分钟回顾                      │
  └─────────────────────────────────┘
            ↓
    累计完成 N 个功能点？
       ├─ 否 → 下一个功能点
       └─ 是 → 进入第三步                 ← 收束节点 → 档位二：收束汇报

三、收束（功能批次闸门）                            ← 产出 → 档位二收束汇报
  ┌─────────────────────────────────┐
  │ 阶段1: 文件与代码整理              │
  │ 阶段2: 集中测试                    │
  │ 阶段3: 规范合规审计（AI + 人双线）  │
  │ 阶段4: 效果验证                    │
  │ 产出: 收束报告 → 更新 STATUS.md    │
  └─────────────────────────────────┘
            ↓
     下一个功能点（下一批次）
```

> 汇报两档贯穿全流程：每个功能点→功能点汇报（默认对话内联、不落盘）、收束节点→收束报告（落盘）。详见 `07-汇报.md`。
```

---

## 核心原则

1. **不越界** — 一次只做一个功能点，不扩展到其他范围
2. **不黑盒** — 每个功能点必须留下可观测的客观证据，不开盲盒
3. **不断档** — 每个功能点完成时更新所有记录文件，下一个人能无缝接手
4. **不拖欠** — 回顾中发现的技术债记录到 STATUS，下个循环优先处理
5. **不积压** — 到达计划预设的收束节点必须执行收束（整理代码、审计合规、验证效果）。收束节点是计划时预设的硬闸门：到点未收束，**AI 拒绝开始下一个功能点**（人不必手动卡，AI 自动拦截，但收束本身仍由人触发执行）
