# AI 协作开发流程 — 汇报

> 汇报 = **基础文件更新**（必做）+ **报告生成**（按需）。每个功能点完成后 AI 必须执行基础文件更新。
> 模板权威：`docs/templates/worklog模板.md`、`docs/templates/汇报模板.md`
> 更新: 2026-07-11

---

## 一、必做：基础文件更新

每个功能点完成后，AI 必须更新以下文件——这是汇报的核心，不可跳过：

| 文件 | 更新内容 | 模板 |
|------|---------|------|
| `worklogs/YYYY-MM-DD_描述.md` | 做了什么 / 验证结果 / 关键决策 / 交接 | `worklog模板.md` |
| `STATUS.md` | 功能点状态 → 已完成 + 完成日期 | `STATUS模板.md` |
| `meta/FILE_GRAPH.md` | 如有文件增删移动则同步目录树（非必做，仅变时更新） | — |

---

## 二、按需：报告生成

基础文件写完后，按需汇报：

```
每个功能点完成 → 功能点完成汇报：默认在对话里内联输出，不落盘
收束节点完成   → 收束报告：落盘到 docs/reports/收束报告-v*.md
```

> **功能点级汇报只在对话里说**（参考 `汇报模板.md` 的"精简格式"），**不单独建 `docs/reports/` 文档**；仅当人明确要求留档时才落盘为 `docs/reports/功能点{编号}-*.md`。
> 报告是基础文件的"对外提炼"，不重复记录事实。数字始终以 `STATUS.md` / `worklogs/` 为准。

---

## 三、二档速查

| 档位 | 触发 | 文件名 | 内容 |
|------|------|--------|------|
| **功能点完成** | 基础文件写完后 | **对话内联**（默认不落盘；人要求留档才建 `docs/reports/功能点{编号}-{简短名}.md`） | 做了什么 / 验证结果 / 变更影响 / 遗留 |
| **收束报告** | 收束节点完成 | `docs/reports/收束报告-v{版本号}.md`（必落盘） | 整理报告 + 测试报告 + 审计报告 + 效果验证 + 技术债+开发清单更新 |

---

## 四、AI 填写指引

> 原则：**AI 填数据，人做判断**。

| 档位 | AI 自动填充 | 必须人工确认 |
|------|-----------|-------------|
| 功能点完成 | 名称、编号、关联 BDD、变更影响 | 验证结果（客观证据）、遗留/风险 |
| 收束报告 | 整理报告、测试数据、AI 审计摘要 | 人审计结论、效果验证、主观质量判断 |

---

## 五、汇报链路

```
基础文件更新（必做）
  worklog / STATUS
        │
        ├──提取──→ 功能点完成汇报（对话内联，默认不落盘）
        │
        └──收束节点──→ 收束报告（落盘 docs/reports/）
```
