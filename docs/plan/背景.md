# 项目背景：通用开发规范

> 更新: 2026-07-11

## 项目定位

一套适用于各类软件项目的通用开发规范，覆盖架构、代码、Git、API、测试、文档六个核心维度，外加 AI 协作开发流程。目标是降低新项目启动成本，统一团队开发语言。

## 当前问题

- 每次新项目启动时，没有统一的规范基准可以参考
- 团队风格不统一，Review 成本高
- AI 辅助开发时缺乏结构化流程指引
- 缺乏可视化的项目状态追踪

## 设计决策

### 为什么是六个维度

- **架构**：先定骨架，再填血肉
- **代码**：日常产出最大，最容易形成风格差异
- **Git**：协作基础设施，出问题影响所有人
- **API**：前后端/服务间契约，改起来代价最大
- **测试**：质量底线，越早规范越省成本
- **文档**：知识的复利，不做就会不断重复解释

### 为什么是 Markdown

- 纯文本，Git 友好，diff 可读
- 任何编辑器都能打开
- 易于生成 HTML/PDF 等多格式输出
- AI 模型天然擅长处理 Markdown

## 现状

7 份文档已完成初稿，当前任务是将项目自身按照 AI 协作开发流程搭建完整的基础设施（STATUS、CLAUDE.md、dashboard、worklogs），实现「吃自己的狗粮」。
