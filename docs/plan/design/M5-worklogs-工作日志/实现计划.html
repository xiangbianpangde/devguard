红线 1 · commit-msg 必带 worklog 引用

触发：每次 git commit（全量 commit）。

处理：check_worklog_ref.py 强制正则匹配，命中则接受、未命中则拒绝。

结果：全量 commit 命中（无拦截事件）。

红线 2 · worklog 模板合规

触发：每份新 worklog 创建时人工审阅。

处理：五段式骨架（做了什么 / 验证 / 决策 / 问题 / 交接）；必填 2 段（做了什么 + 验证结果）不可缺。

结果：31/31 必填段齐全；无工具拦（建议审查时人工检查）。

红线 3 · ADR 路径合规

触发：每份 ADR 创建 / 迁移时。

处理：ADR 0006 固化路径 worklogs/decisions/；任何漂移到 docs/decisions/ 立即回退。

结果：6/6 全部位于 worklogs/decisions/。

红线 4 · 命名规约

触发：每份 worklog 创建时。

处理：钩子正则 worklogs[\\/]+\d{4}-\d{2}-\d{2}[_\-\w]*\.md 约束。

结果：31/31 全部匹配 YYYY-MM-DD_<描述>.md。

红线 5 · 断档补登（流程教训）

触发：V0.2-V1.4 期间（约 13 个收束节点）worklog 累积缺漏。

处理：2026-06-08_v15-backfill-v02-v14-worklogs.md 单份巨型 worklog 一次性回填。

教训：每 3 功能点必须收束（AI 主动拦截，到点未收束拒绝下一功能点）。
