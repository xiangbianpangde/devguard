## 二、阶段 V0.x 起步：建立 worklogs/ 目录（2026-05-26）

### 目标

把"每天做了什么"沉淀为可追溯的 Markdown 日流，配合 6 维度规范正文 + 模板化改造。

### 实现路径

```
worklogs/ 目录不存在
    │
    ▼
创建 8 份 worklog（2026-05-26 一日内）
    ├── 2026-05-26_基础设施搭建.md            # 仓库初始化
    ├── 2026-05-26_基础设施修正.md            # 修正：路径 / 文件名
    ├── 2026-05-26_02-06模板改造.md            # 6 维度规范模板化
    ├── 2026-05-26_模板化改造.md              # 模板体系
    ├── 2026-05-26_模板规约与汇报模板.md       # 模板规约
    ├── 2026-05-26_二轮审查修复.md            # 自我审查 + 修复
    ├── 2026-05-26_文件图谱与仪表盘优化.md     # dashboard 雏形
    └── 2026-05-26_示例代码与仪表盘.md         # src/ 落地
    │
    ▼
docs/templates/worklog模板.md（首次定型五段式骨架）
```

### 改动

| 模块 | 改什么 | 状态 |
|------|--------|:---:|
| `worklogs/` | 新建 8 份 worklog | ✅ |
| `docs/templates/` | 新建 `worklog模板.md` | ✅ |
| `conventions/06-documentation` | worklog 模板索引 | ✅ |

### 关键发现

- 单日多份 worklog 用下划线后缀区分（如 `_二轮审查修复` 是当日第二份）

## 四、阶段 V0.x 重构：decisions/ 移入 worklog（2026-05-27/28）

### 目标

把 ADR 从 `docs/decisions/` 移入 `worklogs/decisions/`，固化"ADR 仅收束节点产出"原则。

### 实现路径

```
原 docs/decisions/ADR NNNN-*.md
    │
    ▼ (迁移)
worklogs/decisions/ADR NNNN-*.md     # ADR 0006
    │
    ▼
新增 4 份 ADR
    ├── ADR 0003 AI 协作流程独立成文
    ├── ADR 0004 规范移入 conventions 目录
    ├── ADR 0005 长文件拆文件夹
    └── ADR 0006 仅在约束节点通过 worklog 整理并进行决策总结   # ← 关键决策
    │
    ▼
同步更新
    ├── FILE_GRAPH.md（决策树更新：ADR 落点 → worklogs/decisions/）
    ├── CLAUDE.md（目录结构同步）
    └── ai-workflow/06-第三步_收束节点.md（补充"决策整理"环节）
```

### 改动

| 模块 | 改什么 | 状态 |
|------|--------|:---:|
| `worklogs/` | 新建 3 份 worklog（含 v3.0 双图谱重构） | ✅ |
| `worklogs/decisions/` | 迁移 5 份 ADR + 新建 ADR 0006 | ✅ |
| `docs/decisions/` | 目录废弃 | ✅ |
| `meta/FILE_GRAPH.md` | 决策树 ADR 落点更新 | ✅ |
| 根 `CLAUDE.md` | 目录结构同步 | ✅ |

### 关键决策（ADR 0006 内容）

- **背景**：决策碎片化 + 与工作流脱节 + 过早形式化 + 重复记录
- **决策**：`decisions/` 移入 `worklogs/decisions/`；仅收束节点产出 ADR
- **理由**：决策质量更高 + 减少噪音 + 可追溯性增强 + 工作流统一
- **后果**：操作变更（worklog「关键决策」段） + 文件变更 + 历史 ADR 路径迁移

## 九、与红线的关系

V2.0.1 时点，本模块累计触发并解决的红线：

| 红线类型 | 触发次数 | 处理方式 |
|---------|:---:|---------|
| commit-msg 必带 worklog 引用（红线 6） | 全量 commit 命中 | `check_worklog_ref.py` 强制 |
| worklog 模板合规 | 31/31 必填段齐全 | 人工审阅（无工具拦） |
| ADR 路径合规 | 6/6 在 `worklogs/decisions/` | ADR 0006 固化 |
| 命名规约 | 31/31 匹配 `YYYY-MM-DD_<描述>.md` | 钩子正则约束 |
| 断档补登 | 1 次 | V1.5 一次性回填 V0.2-V1.4 |

- `decisions/` 子目录按主题再分（如 `decisions/architecture/`、`decisions/process/`）
