## 三句话读懂

`meta/` 是 devguard 的**文件治理中枢** —— 整个仓库的"目录结构 + 引用关系 + 新文件归位"三件事的唯一权威：

1. **单文件承载三大职能** —— 仅 1 份 `FILE_GRAPH.md`，却同时充当**目录树**、**引用关系图谱**、**新文件决策树**三份文档
2. **13 个文件类型分支** —— 决策树覆盖从规范正文、BDD、ADR 到 worklog、模板、脚本的 13 种文件类型
3. **与 CLAUDE.md 互为索引** —— `CLAUDE.md` 的"目录索引"段就是 FILE_GRAPH 的精简版，两边修改时需同步

## 与其他模块的关系

- **被引用**：`CLAUDE.md`（M7）"目录索引"段几乎原样摘抄本图谱的 §一 目录树
- **被引用**：`README.md`（M7）"目录结构"段也是本图谱的简化版
- **约束对象**：所有新增/移动文件前必须查本图谱决策树
- **同步对象**：本图谱与 `CLAUDE.md`、`README.md`、`STATUS.md`（M7）的目录结构必须一致
- **不约束**：M1 `conventions/` 规范正文（规范说"应该怎么做"，本图谱说"文件应该放哪"——两件事正交）
