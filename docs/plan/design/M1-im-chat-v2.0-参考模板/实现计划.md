## 二、阶段 A：私聊入口修复（~4-6h）

### 目标

新用户打开 AgentHub → 左侧面板有「发起私聊」按钮。已有会话的用户 → 启动时自动加载历史会话。

### 实现路径

```
chatStore 新增 hydrateFromSessions
    │  把后端 SessionOut[] 映射成 conversation 对象
    │  以 agent_id 为 key 去重（本地优先）
    │
App 启动时调用 → sessionsApi.list({type:'private'}) → hydrate
    │
LeftPanel 空态改造
    │  dmList.length === 0 → 显示「发起私聊」按钮
    │  点击 → 弹出选择 Agent 对话框 → 创建会话 → 进入聊天
```

### 前端改动

| 模块 | 改什么 | 估时 |
|------|--------|:----:|
| chatStore | 新增 `hydrateFromSessions(sessions)` + sessionIds 映射 | 2h |
| AppShell / App | 启动时调 `sessionsApi.list` + `hydrate` | 1h |
| LeftPanel | 空态加「发起私聊」CTA + 触发 StartChatModal | 2h |

### 后端改动

**无**。`GET /api/sessions?type=private` 已存在。

### 验收

浏览器：新用户 → 看到「发起私聊」按钮 → 点击 → 选 Agent → 进入聊天。刷新页面 → 会话还在列表中。

## 三、阶段 B：会话归档（~6-8h）

### 目标

右键会话 → 归档 → 移入「已归档」折叠分区。刷新页面状态不丢。

### 实现路径

```
Session 实体加 archived: bool = False
    │
    ▼
SessionModel 加列 → 新 alembic migration
    │
    ▼
UpdateSessionCommand 加 archived 字段 → session_service.update 透传
    │
    ▼
sessions.py PATCH 透传 archived → 前端 sessions.ts patch body 加 archived
    │
    ▼
chatStore 加 setConversationArchived + Conversation 类型加 archived
    │
    ▼
LeftPanel 加归档按钮 + 「已归档」折叠分区 + filteredDmList 默认过滤 archived
```

### 改动模块

| 层 | 模块 | 改什么 | 估时 |
|----|------|--------|:----:|
| 后端 L2 | session.py | 实体加 `archived: bool = False` | 0.5h |
| 后端 L1 | models.py | SessionModel 加 column | 0.5h |
| 后端 L1 | 新 migration | alembic 加列（接当前 head） | 0.5h |
| 后端 L3 | commands | UpdateSessionCommand 加 archived | 0.5h |
| 后端 L3 | session_service | update 透传 archived | 0.5h |
| 后端 L4 | sessions.py | PATCH 透传 archived | 0.5h |
| 前端 | sessions.ts | patch body 加 archived | 0.5h |
| 前端 | chatStore | setConversationArchived + 类型 | 1h |
| 前端 | LeftPanel | 归档按钮 + 折叠分区 + 过滤 | 2h |

### 阻塞

**alembic stamp 冲突**：多 worktree 共享 PG 可能导致 `Can't locate revision`。解决：`UPDATE alembic_version SET version_num='<main 链最新>'` 再 `upgrade head`。

### 验收

浏览器：右键会话 → 归档 → 从主列表消失 → 底部「已归档(1)」出现 → 展开可见 → 右键取消归档 → 回到主列表。刷新页面 → 归档状态保持。

## 四、阶段 C：消息操作 E2E（~3-5h）

### 目标

用 Playwright 在真浏览器里跑通消息操作完整链路并截图存档。

### 实现路径

```
新建 e2e/message-actions.spec.ts
    │
    启动 PG/Redis(docker) + 后端:8000 + 前端:5173
    │
    创建会话 → 发消息 → Agent 返回带代码块+diff+URL 的消息
    │
    逐一验证：
    ├── 回复 → Composer 出现引用
    ├── 复制代码 → 剪贴板有内容
    ├── Pin → 消息被固定
    ├── Diff 渲染绿色/红色行
    ├── 网页预览卡片 → 全屏展开
    └── 重新生成按钮 → disabled 状态
    │
    全流程截图存档
```

### 改动

**无源码改动**。仅新建 `src/frontend/e2e/message-actions.spec.ts`。

### 依赖

真 CLI provider 可用或 mock 流式环境。

### 验收

Playwright 全部通过 + 截图可见每个操作的正确 UI 状态。

## 五、阶段 D：对话式创建 Agent（~10-14h）

### 目标

用户输入自然语言描述 → LLM 抽取 → 预览 → 一键创建 Agent。

### 实现路径

```
后端新增 POST /api/agents/draft-from-chat
    │  接收 { description }
    │  调 LLM（复用协调者凭证）→ 抽取 { name, role, system_prompt, skills }
    │  返回草稿 JSON
    │  **PR-01 评审 Approve 后才能实现**

前端新增 ConversationalAgentCreate 组件
    │  聊天式 UI（复用 Composer/MessageBubble）
    │  多轮对话收集需求 → 调 draft-from-chat → 预览草稿卡片
    │  确认 → 调 createAgent

CreateAgentModal Step 1 加入口
    │  「对话式创建」按钮 → 切换到 ConversationalAgentCreate
```

### 改动模块

| 层 | 模块 | 改什么 | 估时 |
|----|------|--------|:----:|
| — | 04-commands | **PR-01 冻结新端点（前置）** | — |
| 后端 L4 | agents.py | `POST /agents/draft-from-chat` 端点 | 3h |
| 后端 L3 | agent_service | draft 方法 + prompt 模板 | 3h |
| 前端 | ConversationalAgentCreate（新建） | 聊天式 UI | 4h |
| 前端 | CreateAgentModal | Step 1 加入口 | 1h |

### 硬阻塞

1. **LLM 凭证**：需可用 LLM（复用协调者凭证 `reactive_router.py`）。不可用则 mock LLM 先做前端。
2. **PR-01 接口先行**：新端点须先冻结在 `docs/specs/04-commands` + 2 人 Approve 后才能写实现。

### 降级

如果 LLM 不可用、或 PR-01 不通过：**砍掉对话式创建，仅保留表单向导**。表单向导已完整可用。

### 验收

输入"帮我建一个前端代码审查 Agent，关注 React 性能" → LLM 返回草稿 → 预览卡片显示 name/role/prompt/skills → 确认 → 跳转新 Agent 详情页。
