## 三句话读懂

AgentHub 的 IM 聊天核心体验还有 4 个缺口：

1. **私聊死路** —— 新用户打开左侧面板，看不到任何可聊天的 Agent，也找不到入口发起私聊
2. **对话列表不能归档/搜索** —— 几十个会话堆在一起，没有归档功能整理，搜索也只搜本地不调后端
3. **消息操作没端到端验证过** —— 回复、复制代码、Pin、Diff 预览这些按钮代码都写了，但没在真浏览器里跑过一条完整链路
4. **创建 Agent 只能填表单** —— 想用一句话描述需求就让系统自动生成一个 Agent，目前只能一步步选模板、填参数

## 与原版 M1 的关系

本文件夹是 v2.0 重写版，与 `M1-im-chat/`（v0.1）的关系：

- **全量重写**：从"改哪些文件/行号/函数名"视角 → "用户流/数据流/架构决策"视角
- **设计在前，实现在后**：设计讲 WHAT + WHY；实现讲 HOW（阶段·里程碑·依赖）
- **核心方案不变**：Session 回灌、归档全链路、E2E 补证据、对话式抽取
- **文档数量不变**：7 件齐全

原版 `M1-im-chat/` **不动**，作为工程参考存档。
