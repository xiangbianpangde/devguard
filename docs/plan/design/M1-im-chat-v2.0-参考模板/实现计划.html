§2 · 最先做 · 零后端

阶段 A：私聊入口修复（~4-6h）

新用户打开 AgentHub → 左侧面板有「发起私聊」按钮。已有会话的用户 → 启动时自动加载历史会话。零新后端。

✦
改动面最小、产品感最强——复用已有 GET /api/sessions?type=private 接口，App 启动调一次 + store 注入。前端只加 store 方法 + 空态按钮 + 启动钩子。

实现路径

chatStore 新增 hydrateFromSessions │ 把后端 SessionOut[] 映射成 conversation 对象 │ 以 agent_id 为 key 去重（本地优先） │ App 启动时调用 → sessionsApi.list({type:'private'}) → hydrate │ LeftPanel 空态改造 │ dmList.length === 0 → 显示「发起私聊」按钮 │ 点击 → 弹出选择 Agent 对话框 → 创建会话 → 进入聊天

前端改动

模块改什么估时
chatStore新增 hydrateFromSessions(sessions) + sessionIds 映射2h
AppShell / App启动时调 sessionsApi.list + hydrate1h
LeftPanel空态加「发起私聊」CTA + 触发 StartChatModal2h

后端改动

无

GET /api/sessions?type=private 已存在

只需前端调用即可，零后端改动。

✓
验收：浏览器新用户 → 看到「发起私聊」按钮 → 点击 → 选 Agent → 进入聊天。刷新页面 → 会话还在列表中。

右键会话 → 归档 → 移入「已归档」折叠分区。刷新页面状态不丢。

实现路径

Session 实体加 archived: bool = False │ ▼ SessionModel 加列 → 新 alembic migration │ ▼ UpdateSessionCommand 加 archived 字段 → session_service.update 透传 │ ▼ sessions.py PATCH 透传 archived → 前端 sessions.ts patch body 加 archived │ ▼ chatStore 加 setConversationArchived + Conversation 类型加 archived │ ▼ LeftPanel 加归档按钮 + 「已归档」折叠分区 + filteredDmList 默认过滤 archived

阻塞

⚠
alembic stamp 冲突：多 worktree 共享 PG 可能导致 Can't locate revision。解决：UPDATE alembic_version SET version_num='<main 链最新>' 再 upgrade head。
✓
