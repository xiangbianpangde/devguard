# 设计提案：约束与模板强化方案

> 版本: v2.3 | 2026-06-11
> 依据: 2026-06-11 用户提出"每次更新即强制 + 收束硬闸门 + 统一 HTML 模板族"方向
> 关联: `docs/plan/开发清单.md` #48-#53, `STATUS.md` V2.3 立项, 前作 `设计提案-流程强制化方案.md`(v1.0, #36-#46)

---

## 一、背景

### 1.1 前作做到哪了

V2.1 流程强制化(#36-#46)把"汇报/更新 STATUS/更新 worklog"从 AI 自觉升级为**提交时拦截**,落地 13 个 pre-commit 钩子 + 5 阶段 CI。但复盘发现它的强制有三个**结构性边界**:

1. **强制粒度是"提交时",不是"每次更新时"**。在对话里改 STATUS、改开发清单不触发任何检查,只有 `git commit` 那一刻钩子才回头核对。中间过程无人盯。
2. **跨文件同步只覆盖 worklog→STATUS 一条边**(`check_status_updated.py`)。四件套(worklog/STATUS/开发清单/CLAUDE)里另外两件(开发清单、CLAUDE)只靠 CLAUDE 规则约束 AI,**无工具焊死**。
3. **收束闸门仍是软闸门**。CLAUDE 规则让 AI"到点拒绝开下一个",`check_convergence_artifacts.py` 只在 CI 事后查产物存不存在——**没有任何工具阻止硬开第 4 个功能点**,靠 AI 守规则。

### 1.2 核心矛盾

> 前作让"产物落盘了没有"可拦;本作要让"**更新发生的当下**就拦"、"**积压到阈值**就拦"、"**模板/豁免/标签**这些散落约定"也被工具焊死。

同时,产出物本身的形态也要统一——当前汇报/计划/实施设计是 markdown 为主、HTML 模板只有一套 `final-report-template`,缺"计划/实施设计/绘图素材库"的权威 HTML 模板,**产出物格式无强制**。

### 1.3 强制的两层模型(本作关键认识)

工具层面只有有限的"拦截点",决定了"每次更新即强制"只能**近似**,不能绝对:

```
拦截点              触发时机            能否"每次更新就拦"   对谁有效
──────────────────────────────────────────────────────────────────
Claude Code 钩子     AI 每次 Edit/Write   ✅ 真·每次更新        仅 AI;人手改不触发;只能提示/打回
(PostToolUse)        文件的瞬间                                 不能"替人补"另一文件
git pre-commit       git commit 时        ⚠️ 只能提交时拦       人 + AI 都拦;焊在持久产物上
CI                   push / PR 时         ❌ 更晚               兜底
```

**结论**:
- **要可靠强制 → 放 git pre-commit**(焊在持久产物,人/AI 通拦)。
- **要"编辑当下就提醒" → 加 Claude PostToolUse 钩子**(只约束 AI、可绕过,是护栏不是铁闸)。
- 两者组合 = "近似的每次更新强制":AI 改 STATUS 当下即被预警,提交时再被硬拦兜底。

### 1.4 必须先解决的坑:过紧耦合会让强制烂掉

不是每次动 STATUS 都该连改 CLAUDE。只把 ⏳ 翻 ✅、改完成日期,硬要求同步 CLAUDE 就是噪音 → 人/AI 转而滥用 `[skip-*]` 豁免 → 豁免一泛滥强制名存实亡。

**所以耦合必须分粒度**:`新增/删除功能点行` 才触发硬同步,`改状态值/日期` 只 warn。而"豁免一多就失控"本身需要一个**被登记 + 被审计的统一豁免出口**(#51)——这是 #52/#53 每道硬拦截的前置依赖。

## 二、目标

### 可量化目标

| 目标 | 现状 | 目标值 |
|------|------|--------|
| 文档同步焊死的边数(四件套) | 1(worklog→STATUS) | ≥ 3(+STATUS→开发清单、STATUS→CLAUDE) |
| "编辑当下即预警"覆盖入口文件 | 0 | 5(STATUS/CLAUDE/开发清单/worklog/HTML 产出) |
| 权威 HTML 模板族 | 1(汇报) | 4(汇报/计划/实施设计/绘图素材库) |
| HTML 产出物格式有校验 | 否 | 章节/结构级 L1(对标 #40-#42) |
| 豁免标记集中登记 + 审计 | 散落、零登记 | 统一豁免清单 + 提交时强制登记 |
| 收束闸门 | 软(AI 自律) | 硬(pre-commit 算积压 ≥3 无收束则拒提交) |
| 「更新时间」标签 | 部分文件缺 / 有标签不强制刷新 | 全覆盖 + 提交时强制刷新 |

### 非量化目标

- 把"每次更新即强制"做到工具能做的上限(Claude 钩子预警 + git 钩子硬拦的组合)。
- 把收束节点从"靠 AI 记得拒绝"升级为"工具直接拒绝提交"。
- 沉淀可一键复制到新项目的最小约束基线(脚手架),让本套强制随项目模板外溢。
- 不重写现有流程定义,只在检测层 + 模板层补盲区。

## 三、方案

### 3.1 总体架构

```
现有约束层(V2.1,不动)
  ├── 13 pre-commit 钩子 + 5 阶段 CI
  └── conventions/_meta.yaml(真源,render_meta.py 投射钩子/CI/仪表盘)

新增"每次更新即强制"层(本作)
  ├── Claude Code 钩子(.claude/settings.json,PostToolUse)   ← 编辑当下阻断(仅 AI)
  │     └── 改 STATUS/开发清单/CLAUDE/worklog 未补齐相关文件 → 阻断后续编辑
  │
  ├── pre-commit 硬拦(git commit 时,人+AI 通拦)
  │     ├── check_doc_sync.py        ← 🆕 文档依赖清单:改 X 必须同改 Y(分粒度)
  │     ├── check_convergence_gate.py← 🆕 积压 ≥N 无收束则拒提交(软闸门→硬闸门)
  │     ├── check_exemption_log.py   ← 🆕 用了 [skip-*] 必须在豁免清单登记
  │     └── check_updated_tag.py     ← 🆕 有「更新」标签的文件提交时必须刷新日期
  │
  ├── 模板族 + 格式强制(docs/templates/ + L1)
  │     ├── 4 套权威 HTML 模板(汇报/计划/实施设计/绘图素材库)
  │     └── check_html_artifact.py   ← 🆕 HTML 产出物章节/结构校验
  │
  └── 脚手架打包(可复制到新项目)
        └── 最小约束基线:钩子 + CI + _meta.yaml + 模板
```

### 3.2 功能点映射(#48-#53)

| # | 功能点 | 解决 §一 的哪个边界 |
|---|--------|-------------------|
| 48 | 统一 HTML 模板族 + 格式强制 | §1.2 产出物格式无强制 |
| 49 | 基准约束脚手架 | 让本套强制可外溢到新项目 |
| 50 | 交流 agent + 强制措施 | 需求对齐/决策澄清的协作前置 |
| 51 | 豁免登记强制化 | §1.4 豁免失控(#52/#53 前置) |
| 52 | 长程任务流程强制规范 | §1.1#3 软闸门 → 硬闸门 |
| 53 | 「更新时间」标签强制化 | §1.1#1 每次更新即强制(样板) |

### 3.3 分阶段实施(按依赖顺序,非编号顺序)

> 三个强制功能点(#51/#52/#53)是同一套"L3 自律 → L1/L2 强制"工程,依赖关系决定顺序:**先建豁免出口,再用最小点跑通组合模式,最后复制到复杂点**。

#### 阶段 A:豁免登记(#51)—— 所有硬拦截的前置

| 任务 | 检测时机 | 内容 |
|------|---------|------|
| 建 `豁免清单`(落 `meta/`,待 FILE_GRAPH 定文件名) | - | 字段:文件/钩子/豁免标记/谁/何时/为何 |
| 🆕 `check_exemption_log.py` | commit-msg | commit message 含 `[skip-*]` → 豁免清单必须有对应未过期条目,否则拒提交 |
| 审计入口 | CI | 汇总所有 `[skip-*]` 使用,产出豁免审计表 |

**完成后**:任何豁免都留痕、可审计,后续硬拦截可放心加。

#### 阶段 B:更新时间标签(#53)—— 最小独立点,跑通组合模式样板

| 任务 | 检测时机 | 内容 |
|------|---------|------|
| 盘点缺「更新」标签的文件 + 补全 | 一次性 | 入口文件统一加 `> 更新: YYYY-MM-DD` 头 |
| 🆕 `check_updated_tag.py` | pre-commit | staged 的受管文件:(a) 必须含「更新」标签,(b) 标签日期与提交一致;豁免 `[skip-updated]`(走 #51 登记) |
| 🆕 Claude PostToolUse 钩子 | AI 编辑当下 | 改受管文件未改「更新」标签 → 阻断后续编辑直到补齐 |

**完成后**:`PostToolUse 预警 + pre-commit 硬拦 + 豁免登记` 三件套在最小点跑通,成为 #52 的复制样板。

#### 阶段 C:收束硬闸门 + 文档依赖同步(#52 + #48 的同步部分)

| 任务 | 检测时机 | 内容 |
|------|---------|------|
| 🆕 `check_convergence_gate.py` | pre-commit | 提交标某功能点 ✅ 时,从 STATUS/开发清单算"距上次收束已完成数",≥ 阈值(默认 3)且无收束记录 → 拒提交;豁免 `[skip-gate]`(走登记) |
| 🆕 `check_doc_sync.py` | pre-commit | 文档依赖清单:`新增/删功能点行` 改 STATUS → 开发清单 + CLAUDE 当前状态必须同改(硬);`改状态/日期` → 仅 warn |

**完成后**:软闸门变铁闸门;四件套同步从 1 条边扩到 3 条边。

#### 阶段 D:HTML 模板族 + 格式强制(#48 主体)

| 任务 | 内容 |
|------|------|
| 4 套权威 HTML 模板 | 汇报 / 计划 / 实施设计 / 绘图素材库,沉淀 `docs/templates/`,登记 README-模板索引 + FILE_GRAPH |
| 🆕 `check_html_artifact.py` | HTML 产出物章节/结构校验(用结构而非逐字,对标 #40-#42 章节级 L1) |

#### 阶段 E:交流 agent + 脚手架(#50 + #49)

| 任务 | 内容 |
|------|------|
| 交流 agent 定义(#50) | 需求对齐/决策澄清何时触发、产出什么;强制措施:关键决策未对齐则拦开工(挂 CLAUDE 规则 + 可选 pre-commit) |
| 基准约束脚手架(#49) | 把钩子 + CI + _meta.yaml + 模板打包成可一键复制基线,区分"必带"与"可选"层 |

### 3.4 豁免机制(统一走 #51 登记)

本作所有新增钩子的豁免标记**不再裸用**,必须在豁免清单登记:

| 钩子 | 豁免标记 |
|------|---------|
| check_exemption_log.py | (元钩子,不可豁免) |
| check_updated_tag.py | `[skip-updated]` |
| check_convergence_gate.py | `[skip-gate]` |
| check_doc_sync.py | `[skip-docsync]` |
| check_html_artifact.py | `[skip-html]` |

## 四、影响范围

### 涉及模块

| 模块 | 影响 |
|------|------|
| `.claude/settings.json` | 🆕 PostToolUse 钩子(编辑当下阻断)——需 update-config |
| `scripts/` | 新增 5 个检查脚本(exemption/updated_tag/convergence_gate/doc_sync/html_artifact) |
| `tests/conventions/` | 新增对应 L4 测试 |
| `conventions/_meta.yaml` | 扩展钩子投射 + 文档依赖清单字段 |
| `.github/workflows/ci.yml` | 新增豁免审计 + 收束闸门 CI step |
| `.pre-commit-config.yaml` | 新增 local hook(由 _meta.yaml 投射) |
| `docs/templates/` | 新增 4 套 HTML 模板 + 登记 README-模板索引 |
| 全体入口文件 | 补「更新」标签 |
| `CLAUDE.md` | 更新当前状态 + 引用关系(文档依赖清单) |
| `meta/FILE_GRAPH.md` | 豁免清单 / 模板族 / 脚手架的归类 |

### 不涉及

- 规范正文(conventions/01-08)— 内容不动
- 流程文档(ai-workflow/)— 流程定义不动
- 前作钩子(#36-#46)— 不重写,只在其上叠加

### 与前作的边界

> 前作 = "产物落盘了没有"在提交时拦;本作 = "更新当下"预警 + "积压/豁免/标签/模板"焊死。两者叠加,不替换。

## 五、风险

| 风险 | 概率 | 缓解 |
|------|------|------|
| Claude PostToolUse 钩子只约束 AI、人手改绕过 | 高(固有) | 明确定位为"护栏非铁闸";真强制始终由 pre-commit 兜底 |
| 文档依赖耦合过紧 → 频繁误拦 → 滥用豁免 | 高 | §1.4 分粒度:结构性改动硬拦、微调只 warn;先 #51 建登记再加硬拦 |
| 收束硬闸门误拦正常提交(如纯文档微调也被算积压) | 中 | 闸门只在"标功能点 ✅"的提交触发,非每次提交;阈值可配 |
| 「更新时间」标签全量补全触发大面积 diff | 中 | 分批补;`check_updated_tag` 先 warn-only,稳定后升 error |
| HTML 模板族体量大、绘图素材库范围模糊 | 中 | 先定 4 套模板的"必备章节契约",素材库 MVP 先收已用素材 |
| 脚手架打包与本项目耦合,复制到新项目水土不服 | 中 | 区分"必带/可选"层;先在本项目 dogfood 再外提 |
| settings.json 钩子配置错误导致每次工具调用报错 | 低 | 走 update-config skill,先在非阻断模式验证 |

## 六、Owner 决策(2026-06-11 已锁定)

| # | 决策点 | 选定 | 对方案的影响 |
|---|--------|------|-------------|
| 1 | 收束硬闸门阈值 | **默认 3 个功能点** | `check_convergence_gate.py` 阈值=3,可配 |
| 2 | 豁免清单位置 | **`meta/`** | 豁免清单落 `meta/`(待 FILE_GRAPH 登记具体文件名) |
| 3 | PostToolUse 钩子强度 | **阻断** | 改受管文件未补齐相关文件 → **阻断后续编辑**,不只预警 |
| 4 | HTML 模板族 | **4 套一次做齐** | 汇报/计划/实施设计/绘图素材库同批交付 |
| 5 | 实施顺序 | **按依赖序 A→E** | 不按编号序;A 豁免登记打前置 → B 标签样板 → C 闸门+同步 → D 模板 → E agent+脚手架 |

> 决策 3 的边界提醒(§1.3 固有限制不变):PostToolUse 阻断**只对 AI 生效**,人手改文件仍绕过;真·人机通拦始终由 pre-commit(git commit 时)兜底。"阻断"是把 AI 侧护栏从"提示"升级为"补齐前不放行",不改变"人手改绕过"这一事实。
