01 · 模块定位

它是什么 / 不是什么

M7 是 devguard 仓库的项目门面——根目录下严格 9 个固定入口文件，分别承担：人类第一印象 / AI 自动加载 / 进度数据 / 治理承诺 四类职责。

它是什么

  • · 人类访客的第一印象（README + 4 件 GitHub 标准件）
  • · AI Agent 的自动加载上下文（CLAUDE.md）
  • · 进度数据的人机共享（STATUS + dashboard）
  • · 治理标准的对外承诺（SECURITY / SUPPORT / CONTRIBUTING / LICENSE）

它不是什么

  • · 不是真源中心 —— 规范真源在 conventions/_meta.yaml（M1）
  • · 不是项目配置 —— pre-commit / CI / 钩子配置不在根目录
  • · 不是文档库 —— 文档库是 docs/（M2），根目录不放散文件
  • · 不是产品结构 —— src/<规范维度>/（M3）按规范维度组织

05 · 核心决策 · 人 vs AI 双入口的强分离

CLAUDE.md ≠ README.md —— 一份文档不能两用

worklog 2026-05-27_CLAUDE与README角色分离.md 记录了一次独立功能点 #30，把"双入口分离"从一次性发现变为永久约束。

CLAUDE.md = AI 专用：仅供 AI Agent 在新会话中自动加载
README.md（所有）= 人类专用：供人类开发者导航和理解项目结构 —— worklog 2026-05-27 · 功能点 #30

一份文档两用 · 失效

  • 受众：AI 扫描 + 人顺序读，风格难两全
  • 行为规则：散在文档中，AI 抓取不准
  • 改 AI 行为时：人读 README 时被噪音干扰
  • 改人话风格时：机器友好结构破坏人体验
  • 工具识别：Claude Code 不识别通用 AI.md
→

双件套分离 · 当前选择

  • 受众：AI 找路由 + 人看导航，互不干扰
  • 行为规则：CLAUDE.md 是 AI 唯一权威上下文
  • 改 AI 行为时：改 CLAUDE 不影响 README
  • 改人话风格时：改 README 不破坏 AI 加载
  • 工具识别：默认自动加载 CLAUDE.md，零配置

06 · dashboard.html 必须 HTTP 启动

file:// 协议 ≠ HTTP 协议

dashboard.html 内部调用 fetch('STATUS.md') 解析进度数据。file:// 协议下浏览器禁止 fetch 本地文件，所以必须通过 HTTP 服务启动。

错误 · file:// 直接双击

// file:// 协议下： fetch('STATUS.md') .then(...) // 浏览器 CORS 拒绝 → 空白 / 404 // 进度卡片全部为 0

→

# Windows：双击脚本 scripts/打开仪表盘.bat # 跨平台： python scripts/start_server.py # 默认端口 8000 # 浏览器自动打开 http://localhost:8000
