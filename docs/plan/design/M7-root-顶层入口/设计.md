## 一、模块定位

### 它是什么

`M7` 是 devguard 仓库的**项目门面**——根目录下严格 9 个固定入口文件，负责：

- 人类访客的**第一印象**（README + 4 件 GitHub 标准件）
- AI Agent 的**自动加载上下文**（CLAUDE.md）
- 进度数据的**人机共享**（STATUS + dashboard）
- 治理标准的**对外承诺**（SECURITY / SUPPORT / CONTRIBUTING / LICENSE）

### 它不是什么

- **不是真源中心** —— 规范真源在 `conventions/_meta.yaml`（M1），CLAUDE.md 只是把"速查表"复制到顶层方便 AI 一次加载
- **不是项目配置** —— pre-commit / CI / 钩子配置不在根目录（除 `dashboard.html` 这个被解析产物）
- **不是文档库** —— 文档库是 `docs/`（M2），根目录不放散文件
- **不是产品结构** —— `src/<规范维度>/`（M3）按规范维度组织，不是按产品模块

## 四、人 vs AI 双入口的强分离（核心决策）

### 4.1 决策来源

worklog `2026-05-27_CLAUDE与README角色分离.md` 记录了一次独立的功能点 #30：

> **CLAUDE.md = AI 专用**：仅供 AI Agent 在新会话中自动加载
> **README.md（所有）= 人类专用**：供人类开发者导航和理解项目结构

### 4.2 为什么强制分离

| 角度 | 一份文档两用 | 双件套分离（当前选择） |
|------|------------|----------------------|
| 受众阅读模式 | AI 扫描 + 人顺序读，风格难两全 | AI 找路由 + 人看导航，互不干扰 |
| 行为规则位置 | 规则散在文档中，AI 抓取不准 | CLAUDE.md 是 AI 唯一权威上下文 |
| 改 AI 行为时 | 人读 README 时被噪音干扰 | 改 CLAUDE 不影响 README |
| 改人话风格时 | 机器友好结构破坏人体验 | 改 README 不破坏 AI 加载 |
| 自动化工具识别 | Claude Code 工具不识别通用 AI.md | 工具默认自动加载 `CLAUDE.md`，零配置 |

### 4.3 落地操作

| 改动 | 改哪个 |
|------|--------|
| AI 行为规则 / 流程图 / 任务路由 | `CLAUDE.md`（**不要**改 README） |
| 项目说明 / 快速开始 / 目录树 / FAQ | `README.md`（**不要**改 CLAUDE） |
| 互指条款 | 顶部各放一句："AI Agent 请阅读 CLAUDE.md" / "人类请读 README.md" |

### 4.4 验证

worklog 2026-05-27 验证结果：

- 13 个 README 全部更新 ✅
- CLAUDE.md 含完整 AI 导航和行为规则 ✅
- 全仓再无 `docs/decisions/` 残留引用（worklogs 历史记录除外）✅
- CLAUDE.md ↔ README.md 角色声明互指 ✅

## 五、dashboard.html 必须 HTTP 启动

### 5.1 为什么不能 file:// 打开

```js
// dashboard.html 中的 fetch 调用
fetch('STATUS.md')
  .then(r => r.text())
  .then(parse)
```

- `file://` 协议下浏览器对 `fetch()` 的同源策略**禁止**读取本地文件
- 双击 `dashboard.html` 打开时显示空白或 404

### 5.2 正确启动方式

| 平台 | 命令 |
|------|------|
| Windows | 双击 `scripts/打开仪表盘.bat`（自动启动 + 打开浏览器） |
| 跨平台 | `python scripts/start_server.py`（默认端口 8000） |

### 5.3 文档明示

README.md 常见问题第 1 条：

> **Q: 怎么打开仪表盘？**
> A: 双击 `scripts/打开仪表盘.bat`（自动启动服务并打开浏览器），或运行 `python scripts/start_server.py`。**不能**直接用 `file://` 协议打开（dashboard 需要通过 HTTP 请求读取 `STATUS.md`）。
