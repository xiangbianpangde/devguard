- **看什么**：项目门面；不是真源中心 / 项目配置 / 文档库 / 产品结构
- **想什么**：为什么 9 文件不全放 `conventions/`？—— 因为这是"项目门面"（人/AI/外部访客），而 `conventions/` 是"项目内部规则源"

### §二 9 文件总览
- **看什么**：5 GitHub 标准 + 4 项目特色
- **易踩坑**：CODEOWNERS 不在 9 文件之内（在 `.github/CODEOWNERS`），不要把 9 文件凑成 10 件

### §三 9 文件逐个介绍
- **看什么**：每个文件的角色 / 受众 / 写入者 / 真源 / 关键约束
- **核心理解**：9 文件之间**互不重位**——CHANGELOG 不写安全策略，SECURITY 不写支持渠道，CONTRIBUTING 不写许可
- **易踩坑**：README 和 CLAUDE.md 都不要写"如何贡献"——留给 CONTRIBUTING.md（避免双源）

### §四 人 vs AI 双入口
- **看什么**：worklog 2026-05-27 决策 + 强制分离的 5 个理由
- **核心理解**：CLAUDE.md 是 AI **唯一**权威上下文；改 AI 行为只改 CLAUDE
- **想什么**：为什么不用 `AI.md`？—— 沿用 Claude Code 工具默认加载名，0 配置生效

### §五 dashboard HTTP 启动
- **看什么**：file:// 协议下 fetch 失败 → 必须 HTTP 启动
- **核心理解**：dashboard.html 自己是**渲染产物**，**唯一**真源是 STATUS.md
- **易踩坑**：手改 dashboard 会被 render.py 覆盖；正确做法是改 STATUS.md 触发重渲染

### §六 GitHub 标准 5 件套
- **看什么**：README / CHANGELOG / SECURITY / SUPPORT / LICENSE / CONTRIBUTING
- **想什么**：GitHub 仓库默认建议 LICENSE 选 MIT 是否过宽？—— 匹配"通用开发规范"复用定位

### §七 边界与降级

## 思考题

1. **9 个文件之间为什么互不重位？** —— 因为每个文件对不同受众/不同场景/不同真源——重位会引入"哪个为准"的歧义。
2. **dashboard 是不是"真源"？** —— **不是**——它是被 STATUS.md 解析出来的渲染产物，**自身**不是真源。这是为什么手改 dashboard 一定会被覆盖。
3. **LICENSE 选 MIT 是否过宽？** —— 不宽——通用开发规范的复用定位要求最大化自由度；MIT 商用/修改/分发/转闭源全允许。
4. **为什么 CODEOWNERS 不在 9 文件之内？** —— 因为 GitHub 实际扫描 `.github/CODEOWNERS` 或根目录的 `CODEOWNERS`；我们放在 `.github/` 是为了与 `.github/workflows/ci.yml` 同位。
5. **如果新加一个 `INSTALL.md`，怎么办？** —— **不能加**——FILE_GRAPH §三决策树明文"⚠️ 除这些既定入口外，根目录不接受新散文件"。安装步骤合并到 README 5 步快速开始。
6. **CLAUDE.md 与 conventions/CLAUDE-规范导航.md 的关系？** —— 根 CLAUDE.md 是"项目级"入口（含 7 条核心规则 + 任务速查），conventions/CLAUDE-规范导航.md 是"规范级"入口（含 17 篇规范路由）；两者**不重复**——前者讲"怎么用本项目"，后者讲"怎么用本规范"。

## 易踩坑速查

| 症状 | 真因 | 修复 |
|------|------|------|
| dashboard 双击打开空白 | `file://` 协议下 fetch 失败 | 跑 `start_server.py` 或 `打开仪表盘.bat` |
| dashboard 改了不被保存 | 是渲染产物 | 改 STATUS.md 触发 `render.py` 重生成 |
| AI 读了 README 不知道改 AI 行为 | 改错文件 | 改 AI 行为**只**改 CLAUDE.md |
| CHANGELOG 不被 GitHub 渲染 | 格式不对 | 遵循 Keep a Changelog 1.1.0 |
| SECURITY 漏洞报告被公开 | 没明文禁止 | SECURITY.md 顶部强调"勿在 GitHub Issues 公开" |
| 根目录出现散文件 | 没查 FILE_GRAPH | 先查 `meta/FILE_GRAPH.md` §三决策树 |
| 9 文件名大小写错乱（`Readme.md`） | GitHub 不识别 | 严格按文件列表的拼写 |
| `LICENSE` 改成 `LICENSE.md` | 部分工具只认 `LICENSE` | 保持 `LICENSE` 无后缀 |
| `CODEOWNERS` 放根目录 | 不影响功能但和 `.github/` 不一致 | 统一放 `.github/CODEOWNERS` |
| `dashboard.html` 改 CSP 头 | 那是 V5.4 设计决策 | 不要手改 CSP，会被 render 覆盖 |
| 添加新治理类规范（如 `CODE_OF_CONDUCT.md`） | 没入 `_meta.yaml` | 先在 `conventions/_meta.yaml` 加 18-code_of_conduct 条目 |
| 收束报告找不到根目录文件 | 报告在 `docs/reports/` 不在根 | 报告与根文件分离 |
