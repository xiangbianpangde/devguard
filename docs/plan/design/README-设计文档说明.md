# 设计文档 (docs/plan/design/)

> **👤 人类参考** | 对应规范：`conventions/06-documentation_文档规范.md`
> 模板：`docs/templates/README-模板索引.md`

存放**复杂功能的设计文档**（涉及 3+ 模块、重大架构变更、需求需多方对齐时）。

简单 CRUD、纯 UI 调整、已有明确方案的小改动**无需**写设计文档。

---

## 何时写 / 何时不写

| 需要写设计文档 | 不需要写 |
|--------------|---------|
| 需求不清晰，需多方对齐 | 简单 CRUD 功能 |
| 复杂功能（涉及 3+ 模块） | 纯前端 UI 调整 |
| 重大架构变更 | 已有明确方案的小修改 |

---

## 模板（五段式）

复制以下结构，命名为 `设计文档-{功能名}.md`：

```markdown
# 设计文档：{功能名称}

## 背景
为什么要做？当前有什么问题？

## 目标
做完后达到什么效果？（可量化）

## 方案
技术方案 + 架构图（Mermaid）

## 影响范围
涉及哪些模块/服务/数据库表？

## 风险
已知风险和缓解措施
```

> 架构决策（"为什么选 A 不选 B"）记录为 ADR，放在 `worklogs/decisions/`。ADR 仅在收束节点时创建（见 ADR 0006）。
