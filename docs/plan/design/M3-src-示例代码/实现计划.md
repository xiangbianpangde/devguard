## 四、阶段 V0.x 重构：与规范 §二 1:1 对齐（2026-05-27/28）

### 目标

V0.x 起步时各子目录的内容是"独立创作"的，V0.x 重构时强制要求：**src/ 内的每个配置文件都要在对应规范的 §二「落地」段找到原文**。

### 实现路径

```
conventions/01-08 规范 §二「落地」段
    │
    │ 1:1 映射
    ▼
src/<维度>/ 配置文件
    │
    │ 改 src/ 必同步 worklog
    ▼
worklogs/<日期>_规范§二对齐.md
```

### 关键对齐点

| 维度 | §二 关键内容 | src/ 落地 |
|------|-------------|----------|
| 01 架构 | 4 层单向依赖 + 领域层禁框架 | importlinter.ini: 2 个契约 |
| 02 代码 | ruff 规则集（E/F/N/T20/S/PLR0915）| ruff.toml: 完整 select + per-file-ignores |
| 03 Git | pre-commit 4 钩子 + commitlint 8 类型 | .pre-commit-config.yaml + commitlint.config.js |
| 04 API | spectral 3 规则（kebab-case/禁动词/安全声明）| .spectral.yaml: 3 条规则 |
| 05 测试 | 覆盖率门禁 ≥ 80% | pytest.ini: --cov-fail-under=80 |
| 06 文档 | Keep a Changelog + Google docstring | CHANGELOG.md + docstring_example.py |
| 08 图谱 | 双图谱范式 | call_graph_example.py（教学） + README 指向真工具 |

### 关键反直觉保留

`naming_bad.py` 故意保留命名违规（用于 L4 规范测试断言），`ruff.toml` 加 per-file-ignores 豁免 —— **教学反模式 ≠ 真实违规**。

### worklog 引用

- `worklogs/2026-05-27_规范正文v2重构.md`
- `worklogs/2026-05-28_08规范v3.0双图谱重构.md`
