01 · 模块定位

它是什么 / 不是什么

M8 infrastructure 是 devguard 的强制约束防线。 把 M1 conventions/ 定义的 34 条红线以可执行形态落到 3 个地方：本地拦截、服务端拦截、规范自检。

它是什么

devguard 强制约束防线的基础设施层，由 86 个组件协同构成：

它不是什么

规范定义：M1 管辖 单文件：5+10+65+6 集合 可选：默认全开

4.2 commit-msg 阶段 · 1 钩子 + commitlint

⑩ commit-msg-worklog-ref local · scripts/check_worklog_ref.py
作用：worklog 引用校验。
阻断示例：commit message 必须引用 worklogs/YYYY-MM-DD_*.md，否则 commit 拒绝。
效果：V1.x 范式跃迁的标志——"无记录提交 = 不能 commit"成铁律。
对应规范：07-ai-workflow §汇报。

commitlint (Conventional Commits)

作用：验证 commit message 符合 Conventional Commits 规范。

阻断示例：type 不在 enum（feat/fix/chore/docs/...）/ subject < 5 字符 / 首字母大写 / 句末句号。

commitlint-pre-commit-hook v9.16.0 stages: [commit-msg]

4.3 双层设计原因

pre-commit 阶段

  • 适用：文件内容相关
  • 原因：此时文件已写入工作区，可读可改可 lint
  • 钩子：9 个（trailing-ws / EOF / yaml / json / large-files / ruff / ruff-format / gitleaks / markdownlint）
⇄

commit-msg 阶段

  • 适用：commit message 相关
  • 原因：commit 还未生成，message 可校验可阻断
  • 钩子：2 个（commitlint + worklog-ref）
!
不能放错阶段：gitleaks 必须 pre-commit（要扫文件内容）；commitlint 必须 commit-msg（要扫 message）。放错阶段 → 钩子永远跑不到对应数据。

M8 v1.0 验收 · 一句话

5 CI 阶段 + 10 钩子 + 65 L4 tests + 6 治理文件 = 86 组件协同强制约束范式

M8 infrastructure 把 M1 conventions/ 的 34 条红线以可执行形态落到 3 个地方： 本地拦截（pre-commit 10 钩子秒级 fail-fast）→ 服务端兜底（CI 5 阶段分钟级 fail）→ 规范自证（L4 65 tests 持续验证）→ 治理闭环（6 治理文件配套）。
