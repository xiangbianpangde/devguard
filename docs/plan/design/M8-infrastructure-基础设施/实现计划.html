关键技术

# .github/workflows/ci.yml · build 阶段 - name: Collect L4 stats run: | L4_STATS=$(python scripts/collect_l4_stats.py 2>/dev/null || echo "L4_STATS=0/0") L4_PASSED="${L4_PASSED:-0}" # bash 防御空字符串 L4_TOTAL="${L4_TOTAL:-0}" echo "L4_PASSED=$L4_PASSED" echo "L4_TOTAL=$L4_TOTAL" - name: Render dashboard run: | python render.py --meta _meta.yaml --status STATUS.md --out dashboard.html git diff --exit-code dashboard.html
!

M8 演进 · 一句话

从 V0.1 的 9 钩子 + 13 L4 基线，演进到 V2.0.1 的 10 钩子 + 65 L4 tests + 8 dogfood 拦截

M8 演进不是"渐进式重构"，而是"收束锚定 + 增量叠加"——V0.1 锚定基线（9 钩子 + 5 阶段 CI + 13 L4）， 后续 6 个 V0.x 增量（V0.2-V0.8）持续叠加（钩子 9→10，L4 13→38），V1.1 钩子化 gitleaks（钩子稳 10）， V2.0.1 dogfood 闭环用 devguard 自己的代码验证 L4 跃迁（38→65）+ 8 拦 + 全修。

三曲线：钩子数阶梯（V0.2 后稳定 10）/ L4 tests持续增长（13 → 65，5 倍）/ CI 阶段稳定（5 阶段不变）。

核心证据：V2.0.1 dogfood 8 拦——违规真被拦，不是"测试通过"。这是 V1.x 范式跃迁的最强闭环证据。
