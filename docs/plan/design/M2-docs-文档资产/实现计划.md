## 六、阶段 V1.5 final-report-template：学术风 HTML 沉淀（2026-06-08）

### 目标

把 V1.x 收尾汇报的"高密度学术风 + 11 Mermaid + 3 Tab + 5 进度条 + 2 数字滚动 + 1 模拟器"沉淀为模板，**作为后续汇报的权威来源**。

### 实现路径

```
用户提出"之后的汇报 HTML 以 V1.x 最终汇报为标准"
    │
    ▼
新建 docs/templates/devguard/final-report-template/
    ├── template.html       118KB 占位符版（41 个 {{占位}} + 11 Mermaid + 3 Tab + 5 进度条）
    ├── render.py           std-lib 替换脚本（兼容 KEY 和 {{KEY}} 两种 JSON 风格）
    ├── example-data.json   V1.x 真实数据，可直接跑
    ├── README.md           12 节骨架说明 + 41 占位符清单 + 改造规则
    └── demo.html           render.py + example-data.json 跑出的实例
    │
    ▼
docs/reports/
    ├── 2026-06-08_devguard_V1x_final_report.html         # 原始带数据版
    ├── 2026-06-08_devguard_V1x_final_report_demo.html    # render.py 渲染版
    └── 2026-06-08_devguard_V1.5_V2.0_merged_report.html  # V1.5/V2.0 合并
    │
    ▼
同步 README-模板索引.md + FILE_GRAPH.md
```

### 关键决策

1. **双花括号 `{{占位}}` 而非单花括号 `{占位}`** —— 单花括号是 docs/templates 的传统约定，但本模板用 `{{双}}` 以避免与 Mermaid 块、JS 字符串里的单花括号冲突。render.py 兼容两种 JSON key 风格
2. **占位符分 8 类**（基础 4 + 周期 4 + KPI 8×3 + 健康度 5×2 = 41）—— 够覆盖 80% 报告场景
3. **不依赖 Jinja2** —— std-lib `str.replace` 即可，新项目 fork 时无额外依赖

| `docs/templates/devguard/final-report-template/` | 新建 5 件（template + render + data + README + demo） | ✅ |

## 十、与红线的关系

| 红线类型 | 触发次数 | 处理方式 |

| 模板与规范不同步（10 规范） | ~5 | 改 conventions 必同步 `templates/devguard/conventions/` |
