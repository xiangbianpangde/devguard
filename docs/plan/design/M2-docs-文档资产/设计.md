## 一、模块定位

### 它是什么

`docs/` 是 devguard 的**文档资产仓库**。除 `conventions/` 中的规范正文外，所有"文档型"内容全部在此：

- **模板**（让新项目能复制）
- **BDD**（让规范有验收）
- **计划**（让功能点有依据）
- **汇报**（让进展可追溯）
- **调研**（让决策有依据）

### 它不是什么

- **不是规范正文** —— 规范在 `conventions/`（M1）
- **不是工作日志** —— 日志在 `worklogs/`（M5）
- **不是元信息** —— 元信息在 `meta/`（M6）

## 三、模板真源策略

### 真源 = `docs/templates/`

| 复制场景 | 复制源 | 目标 |
|---------|--------|------|
| 新项目用 devguard 整套 | `docs/templates/devguard/` | 新项目根目录 |
| 单独用某个模板（如 BDD） | `docs/templates/BDD规格模板.md` | 新项目 `docs/specs/` |
| 仪表盘报告渲染 | `docs/templates/devguard/html-report-template/` | 新项目根 |
| V1.x 学术风总结 | `docs/templates/devguard/final-report-template/` | 新项目报告目录 |

### 模板规约（10 规范）

每个模板必须：
- 在 `docs/templates/README-模板索引.md` 登记
- 内含"占位符"说明（`<项目名>` / `<日期>` 等）
- 与对应规范保持版本同步

## 四、BDD 与 conventions 的互检

```
conventions/01-architecture.md  ←──互检──→  docs/specs/01-architecture.md
                  │
                  │ 改规范 → PR-09 红线触发 → 必须改 BDD
                  ↓
                CI compliance 阶段 check 互检
```

**关键约束**：
1. conventions/NN-*.md 改动 → 同名 BDD 必须同 PR 修改
2. BDD 中"Given/When/Then" 必须可对应到 src/<维度>/ 示例
3. specs/00 是基础设施 BDD，不属于 conventions（项目级）

## 六、汇报体系（三档）

| 档次 | 触发 | 是否落盘 | 模板 |
|------|------|:---:|------|
| 功能点级 | 每个功能点完成 | ❌ 对话内联 | 无 |
| 收束级 | 预设收束节点（默认每 3 功能点） | ✅ `收束报告-vX.Y.md` | `收束报告模板.md` |
| 阶段级 | 跨多收束节点的总览 | ✅ `阶段汇报-<主题>.md` | `汇报模板.md` |
| 最终成果 | 大版本收尾（V1.x → V2.0） | ✅ HTML 高密度学术风 | `final-report-template/` |

## 七、与其他模块的关系

- **依赖**：依赖 `conventions/`（M1）定义"规范"概念
- **被依赖**：`worklogs/`（M5）收束节点产出 → `reports/`；`STATUS.md`（M7）读 `plan/开发清单.md`
- **导出**：`templates/devguard/` → 新项目
- **示例**：`design/M1-im-chat-v2.0-参考模板/` 是 docs/plan/design/ 子目录的模板示范

## 八、边界与降级

| 场景 | 降级行为 |
|------|---------|
| 改规范没改 BDD | CI compliance 失败 + PR-09 红线 |
| 收束节点未产报告 | AI 拒绝开始下一功能点（hardgate） |
| 模板被改但 devguard 子目录没同步 | 10 规范 L1 检测失败 |
| 调研报告无人读 | 不强制；但决策必须能追溯到调研依据 |

### 明确不做

- **不做**：功能点级汇报落盘（避免堆积）
- **不做**：模板放 `src/` 或根目录（必须在 `docs/templates/`）
- **不做**：BDD 编号跳跃（必须连续，缺号说明）

## 九、验收标准

| 维度 | 验收方式 | 通过标准 |
|------|---------|---------|
| 模板完整性 | 人工检阅 `README-模板索引.md` | 14+ 模板全部登记 |
| BDD 互检 | CI compliance | conventions ↔ specs 一对一 |
| 计划同步 | `开发清单.md` 与 `STATUS.md` 比对 | 行数一致 |
| 收束报告 | `INDEX.md` 列表 | 14 份齐全（v0.1-v1.5） |
| HTML 渲染 | `final-report-template/render.py` | V1.x final 报告渲染成功 |
