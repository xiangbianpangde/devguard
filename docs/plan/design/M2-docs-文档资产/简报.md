## 核心决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 模板放哪 | `docs/templates/` 集中，不散落 | 单一权威源；新项目知道复制哪 |
| 是否分通用 vs devguard | 是，`devguard/` 子目录 | 通用模板可单独取用；devguard fork 整组 |
| BDD 编号怎么定 | 与 conventions 对齐（01-08） | 一对一互检，编号易追 |
| 设计文档放哪 | `docs/plan/design/<功能名>/` 子目录 | 长设计需多文件（设计+实现+简报+笔记） |
| 汇报落不落盘 | 功能点级**不落盘**（内联）；收束级**落盘** | 避免堆积无意义流水汇报 |
| 调研深度怎么组织 | 多 N 级目录（N1/N2/N3 ...） | 深度研究产物天然层级化 |
