## 设计.md 章节导航

### §一 模块定位
- **看什么**：它是什么（文档资产仓库） / 不是什么（不是规范/日志/元信息）
- **想什么**：为什么 `docs/` 和 `conventions/` 分开？—— 规范正文（conventions）是"规则源"，其他所有文档（docs）是"规则的实例与衍生"

### §二 五个子模块
- **看什么**：templates / specs / plan / reports / research 各自职责
- **核心理解**：**`templates/` 的"两层设计"**（通用 vs devguard/）—— 通用模板任何项目都能用，devguard 子目录是"打包 fork"
- **易踩坑**：`docs/reports/` 里有 4 份 HTML（V1.x 收尾 + V1.5/V2.0 合并），它们是 final-report-template 的实例，不是模板本身

### §三 模板真源策略
- **看什么**：4 种复制场景的对应源
- **核心理解**：**"新项目知道复制哪"** —— `docs/templates/devguard/` 整组 cp -r，是这个模块最核心的设计判断
- **易踩坑**：单花括号 `{占位}` 是 docs/templates 传统约定，但 final-report-template 用 `{{双}}`，render.py 兼容两种

### §四 BDD 与 conventions 的互检
- **看什么**：conventions/NN ↔ specs/NN 一对一
- **想什么**：为什么 specs/00 是基础设施 BDD，不属于 conventions？—— 基础设施是"项目级"，不是"规范级"

### §五 计划 → 状态 → 仪表盘
- **看什么**：开发清单.md → STATUS.md → dashboard.html 的链路
- **核心理解**：**人写新功能点要同步两个地方**（开发清单 + STATUS）—— 这是 M7 仪表盘的"数据源/视图分离"基础

### §六 汇报体系（三档 + 1 档最终）
- **看什么**：功能点级（不落盘）/ 收束级（落盘）/ 阶段级（按需）/ 最终成果（HTML 高密度）
- **易踩坑**：**功能点级默认对话内联、不落盘**——这是避免堆积无意义流水汇报的关键决策

### §十 红线关系
- **看什么**：本模块累计触发的红线类型
- **易踩坑**：PR-09 spec 同步触发 ~10+ 次 —— 改 conventions 必同步改 BDD

## 思考题

1. **为什么模板要分通用 vs devguard 两层？** —— 通用模板（BDD/README/CLAUDE 等）任何项目都能用；devguard 子目录是"打包 fork 源"，新项目只复制自己需要的子集。两层分离避免"想取一个 README 模板却要 fork 整组规范"的痛苦。
2. **功能点级汇报为什么不落盘？** —— V0.7 收尾定义"汇报三档"时的关键决策：49 个功能点每个都落盘会变成"流水账文档垃圾"，真正可追溯的只有"收束级 + 阶段级 + 最终成果"。功能点级走 worklog → 收束节点 → 收束报告 路径。
3. **BDD 编号为什么不直接从 01 开始？** —— specs/00-infrastructure-template.md 是"项目级"基础设施 BDD（项目骨架），不属于 conventions（规范级）。00 显式标记"项目级"与 01-08 "规范级"的边界。
4. **final-report-template 为什么不依赖 Jinja2？** —— V1.5 沉淀时的关键决策：新项目 fork 时无额外依赖，std-lib `str.replace` 即可；41 个占位符分 8 类够覆盖 80% 报告场景。代价是"无循环/无逻辑"，但通过"双花括号 + render.py 兼容 KEY/{{KEY}}"缓解。
5. **research 子模块的命名变体（N1-...构建方法.md / N1-...的构建方法.md）要不要清理？** —— 当前保留（作为"调研过程产物"的可追溯性），但 V2.0+ 可去重为单一权威命名。

## 易踩坑速查

| 症状 | 真因 | 修复 |
|------|------|------|
| 改规范但 BDD 没改 | 跳过 PR-09 红线 | CI compliance 阶段失败 + 阻断 PR |
| 改了 `docs/templates/devguard/conventions/` 但根 `conventions/` 没同步 | 双向同步遗漏 | 用 diff 比对两边 `_meta.yaml` |
| 收束节点未产报告被 AI 拦截 | hardgate 触发 | 产 `收束报告-vX.Y.md` 后再继续 |
| 新模板没登记 `README-模板索引.md` | 10 规范 L1 检测失败 | 跑 `render_meta.py --check` 看报错 |
| 复制 `final-report-template` 但 `{{占位}}` 被改坏 | render.py 替换语法 | render.py 兼容两种 KEY 风格；保留 `{{}}` 包裹 |
| `dashboard.html` 不更新 | `STATUS.md` 没改 | dashboard 只解析 STATUS.md 表格 |
| `docs/plan/背景.md` 与设计.md 矛盾 | 文档分叉 | 改设计必先改背景（背景是"为什么"，设计是"怎么做"） |
| 调研报告无人引用 | 调研失联 | 决策时显式引用 `docs/research/<主题>/report.md` |
