## 设计.md 章节导航

### §一 模块定位
- **看什么**：它是规则源；不是文档库 / 配置中心 / 示例代码
- **想什么**：为什么 `_meta.yaml` 在 `conventions/` 而不是根目录？—— 因为它和规范正文一体（元数据描述的就是同目录的规范）

### §二 目录结构
- **看什么**：8 个开发规范文件 + 1 个目录（AI 流程）+ 2 个导航 + 1 个真源
- **易踩坑**：编号没有"07-" 单文件 ——`07-ai-workflow` = 整个 `ai-workflow_*/` 目录

### §三 真源 + 渲染策略
- **看什么**：B3 方案选择原因
- **核心理解**：**改 `_meta.yaml` → 钩子/CI/仪表盘全自动同步**——这是整个模块最核心的设计判断
- **易踩坑**："分级标签"小节是渲染产物，不能手改；手改也会被 CI 覆盖

### §四 规范分类
- **看什么**：开发 8 篇（有正文）vs 治理 9 篇（仅元数据）
- **想什么**：为什么治理类不写正文？—— 治理类"存在性即满足"，写长文反而无的放矢

### §五 正文结构
- **看什么**：5 段式（红线/配置/决策/反模式/检查清单）
- **易踩坑**：写新规范时必须严格按这个结构，否则 L4 测试会断

### §六 双入口
- **看什么**：CLAUDE（AI）vs README（人）
- **核心理解**：强制分离避免"一份文档两用"的妥协

### §七 AI 流程模块
- **看什么**：10 篇结构 + v2.0 双轨制
- **想什么**：双轨制为什么重要？—— 研究类任务上"工程级"流程会卡死调研节奏

## BDD 验收清单

实施时如要新增/修改本模块，必须满足：

- [ ] **BDD 1（真源完整）**：`_meta.yaml` 17 条目全部有 `l3_route` + `l1_check` + `l1_check_path`
- [ ] **BDD 2（路由表）**：`CLAUDE-规范导航.md` 表格行数 = 17
- [ ] **BDD 3（红线总数）**：所有规范 §一红线汇总 ≥ 34
- [ ] **BDD 4（双入口分离）**：CLAUDE / README 各自独立，不内容互抄
- [ ] **BDD 5（5 段式）**：01-08 每篇都有完整 5 段
- [ ] **BDD 6（治理类）**：09-17 在 `_meta.yaml` 都有 `l1_check_path` 指向实际文件
- [ ] **BDD 7（AI 流程 10 篇）**：`ai-workflow_*/` 文件数 = 10
- [ ] **BDD 8（去重）**：调用图主题只在 08 出现（其他规范引用而不复制）
- [ ] **BDD 9（互检）**：每篇 conventions 都有对应 `docs/specs/`
- [ ] **BDD 10（渲染同步）**：`render_meta.py --check` 退出码 0
- [ ] **BDD 11（L4 测试）**：`pytest tests/conventions/` 65 passed
- [ ] **BDD 12（CI 5 阶段）**：`compliance` 阶段绿灯

## 思考题

1. **为什么治理类规范不写长文？** —— 因为它们的本质是"checklist 类"约束（文件必须存在 + 格式正确），写长文反而会引导读者去关心"哲学论述"而不是"清单本身"。
2. **`_meta.yaml` 改 schema 时怎么向后兼容？** —— 当前 version 1.0.0，未来加 `deprecated` 字段时如何不破坏现有 17 条目？
3. **AI 流程 10 篇是不是太多了？** —— 长文拆目录 ADR 0005 设了 300 行阈值，但 10 篇之间是不是有过度切分？
4. **`l3_route` 字段会不会成为重复表达？** —— 它同时存在于 `_meta.yaml` 和 `CLAUDE-规范导航.md` 中（后者由前者生成），但 CI 校验时如何确保不漂移？
5. **新增第 18 篇规范的最小工作集是什么？** —— 至少 6 件：`_meta.yaml` 加条 + 正文（如果是开发类）+ BDD + L1 检测脚本 + 模板同步 + worklog。

## 易踩坑速查

| 症状 | 真因 | 修复 |
|------|------|------|
| 改了规范但 dashboard 不更新 | 没跑 `render_meta.py` | 加入 pre-commit 钩子 |
| 改了"分级标签"小节但被 CI 覆盖 | 那是渲染产物 | 改 `_meta.yaml` 的 grade 字段 |
| 新规范没有 L1 检测被 CI 拒 | `l1_check` 字段缺失 | 在 `_meta.yaml` 补全 |
| AI 流程新增第 11 篇但 AI 不知道 | 没更新 `CLAUDE-规范导航.md` | `render_meta.py` 自动同步 |
| 改 `_meta.yaml` 没加 `[meta]` 标记 | commitlint 不报错但难追溯 | 习惯性加 `[meta]` |
