## 二、阶段 V0.x 起步：仪表盘启动器（2026-05-26）

### 目标

解决 dashboard.html 的本地预览问题：浏览器 `file://` 协议禁止 fetch 跨文件读取 `STATUS.md`，必须经由 HTTP 打开。

### 实现路径

```
浏览器直接打开 dashboard.html
    │
    ├── FAIL: fetch('STATUS.md') 跨文件协议阻断
    │
    ▼ 解决
python scripts/start_server.py  →  127.0.0.1:8080
    │
    ▼ Windows 友好
双击 打开仪表盘.bat
```

### 改动

| 文件 | 改什么 | 状态 |
|------|--------|:---:|
| `scripts/start_server.py` | HTTP server + 回环地址 + 端口顺延 | ✅ |
| `scripts/打开仪表盘.bat` | UTF-8 + py 优先 + python 回退 + pause | ✅ |

### 关键设计

- `QuietHandler` 类：安静日志 + `Cache-Control: no-store` 强制不缓存（每次刷新读到最新 STATUS.md）
- 端口冲突自动顺延最多 20 次

### 目标

### 实现路径

```
check_worklog_ref.py（#10 任务）
    │
    ▼ pre-commit commit-msg 钩子
正则匹配 worklogs/YYYY-MM-DD_*.md
    │
    ▼ V6.1 引入

    │ 期望关键词表（7 篇 × 5-7 关键词）

    │ 替代 bash grep：Python 跑 pytest + 解析 L4 数字
    │ 输出 last line 格式: L4_STATS=64/64
```

### 改动

| 文件 | 改什么 | 状态 |
|------|--------|:---:|
| `scripts/check_worklog_ref.py` | 58 行 · WORKLOG_PATTERN 正则 + 豁免标记 | ✅ |
| `scripts/check_ai_workflow.py` | 98 行 · V9.1 章节级 L1 钩子 | ✅ |
| `scripts/check_code_understanding.py` | 66 行 · ast.parse + 关键函数校验 | ✅ |

### 关键设计

- **commit-msg 钩子**：用 pre-commit 框架而非自建，钩子安装/升级标准化

- **AST 解析**：避免动态导入，只查静态函数定义
- **跨平台 L4 统计**：避免 PowerShell/ANSI 编码问题，CI 行为一致

## 十、与红线的关系

V2.0.1 时点，本模块累计触发并解决的红线：

| 红线类型 | 触发次数 | 处理方式 |
|---------|:---:|---------|
| 红线检测脚本自身 L1 失败 | ~3 | V5.2 → V5.3 关键词表升级；V6.1 引入章节级 L1 |
| 渲染漂移 | ~2 | V0.1 加 `meta-render` 钩子；CI `test` 阶段 `--check` 兜底 |
| worklog 引用漏 | ~5 | `check_worklog_ref.py` 拦下，习惯性加引用 |
| L4 统计失败 | ~2 | V6.3 retry 改 Python 解析；V7.2 加默认值防御 |
| markdownlint 误报 | ~5 | V0.6 引入 lint_markdown.py 分批调用，跨平台稳定 |
