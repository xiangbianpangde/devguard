## v2.0 相比 v1.0 的变化

| 维度 | v1.0 | v2.0 |
|------|------|------|
| 任务模型 | 单一线性流程（调研→计划→开发→收束） | **双轨制**：长程任务 + 思考任务 |
| 模块视角 | 无 | **新增**：工程模块 vs 基础+功能模块 分类体系 |
| 设计规范 | 分散在各步骤 | **独立成篇**：7 件套设计文件夹 + 8 节结构模板 |
| 验收标准 | 隐含 | **显式三级**：功能点级 / 收束节点级 / 项目级 |
| 端到端 | 未覆盖完整链路 | **新增**：PRD → BDD+TDD → 技术转换(9文档) → 模块划分 → 并行开发 → 测试 |
| 技术转换 | 无 | **新增**：PRD 到技术语言的 9 项产出 |
| 参考模板 | 无 | 引用 M1-im-chat-v2.0 作为设计/实现范本 |
| 部署规范 | 无 | **新增**：统一服务器（max）+ 模块插入机制 + 部署全流程（门禁→推送→插入→验证→回滚） |

## 核心原则（贯穿全部 9 篇）

1. **不越界**：AI 不替人做产品决策 / 收束决策 / 安全决策
2. **不黑盒**：所有决策可追溯（ADR + worklog + commit）
3. **不断档**：每功能点完成必汇报（worklog + STATUS）
4. **不拖欠**：收束节点硬闸门，到期未收束 → 拒绝开始新功能点
5. **不积压**：汇报即时完成，不堆积到收束节点一次性补
