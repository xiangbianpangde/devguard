# AI 协作开发流程 — 角色分工与文件体系

> 适用场景：使用 AI（Claude/DeepSeek/GPT 等）作为主力开发者的项目。
> 核心理念：先计划后开发、BDD 驱动迭代、可视化验证、持续回顾。

---

## 角色分工

| 角色 | 模型 | 职责 |
|------|------|------|
| **执行者** | Claude | 写代码、修 bug、跑测试、生成文档 |
| **审查者** | DeepSeek | 读 diff、对照计划检查、发现遗漏和风险 |
| **架构师 + 开发** | Opus | 架构设计、开发计划审查、代码审查 |
| **决策者** | 我 | 验收功能、决定方向调整、拍板关键选择 |

> 你只在审查者标记的差异点和高风险项上做决策，不做逐行审查。

---

## 文件体系

```
project/
├── conventions/              # 开发规范（本文件夹）
│   ├── README.md             # 索引入口
│   ├── 01-architecture_*.md  # 架构设计规范
│   ├── 02-coding_*.md        # 代码编写规范
│   ├── 03-git_*.md           # Git 协作规范
│   ├── 04-api_*.md           # API 设计规范
│   ├── 05-testing_*.md       # 测试规范
│   ├── 06-documentation_*.md # 文档规范
│   └── 07-ai-workflow_*/     # AI 协作开发流程（本文件夹）
├── docs/
│   ├── templates/            # 所有文件模板
│   ├── research/             # 调研文档
│   ├── specs/                # BDD 功能规格
│   └── decisions/            # 架构决策记录 (ADR)
├── plan/                     # 计划提纲 + 实时进度
│   ├── 背景.md               # 流程笔记与背景分析
│   └── 开发清单.md            # 功能点开发计划清单
├── STATUS.md                 # 当前状态仪表盘
├── dashboard.html            # 可视化仪表盘（自刷新）
├── CLAUDE.md                 # Claude 项目上下文（新会话自动加载）
├── worklogs/                 # 工作日志
└── src/                      # 代码
```
