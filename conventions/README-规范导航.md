## 红线速查

违反任一条 = **方案打回 / 审查不通过**。完整列表见各规范 §一「红线」。

### 🏗️ 架构（01）
1. 禁止循环依赖 · 2. 领域层不依赖框架/ORM · 3. 禁止跨层调用
（大型项目由 import-linter 契约 + [08 图谱](08-code-understanding_代码理解与图谱规范.md) 自动检测）

### 💻 代码（02）
1. 禁止 `print()`/`console.log` → logging · 2. SQL 参数化，禁拼接 · 3. 密钥禁硬编码 → 环境变量
4. 禁止 `except: pass` 静默吞异常 · 5. 禁止注释代码块 / 调试断点 · 6. 日志脱敏
（1/3/4/5 由 ruff + gitleaks pre-commit 自动拦）

### 🔄 Git（03）
1. `main` 禁直接 push · 2. Conventional Commits · 3. 一 commit 一事 · 4. 禁 force push 共享分支 · 5. ≥1 Approve
（由 commitlint + 分支保护强制）

### 🌐 API（04）
1. 资源名词复数 + kebab-case，无动词 · 2. 统一响应 `code/message/data` · 3. 所有接口默认需认证 · 4. 向后兼容不删字段

### 🧪 测试（05）
1. 测试独立不依赖顺序 · 2. 只 Mock 外部边界 · 3. 覆盖正常+边界+异常 · 4. 无 flaky test

### 📄 文档（06）
1. 必有 README（快速开始）· 2. 发布更新 CHANGELOG · 3. 文档随码同 PR · 4. 公共 API 有 docstring
