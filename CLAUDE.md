# 通用开发规范 — Claude 项目上下文

> 本文件在新会话中自动加载。每次功能点完成后更新。

## 项目概述

一套适用于各类软件项目的通用开发规范，覆盖架构、代码、Git、API、测试、文档六个维度，外加 AI 协作开发流程。所有文件为 Markdown 纯文本，Git 管理。

## 目录结构

```
开发规范/
├── 01-架构设计规范.md        # 分层原则、模块划分、依赖管理
├── 02-代码编写规范.md        # 命名、注释、错误处理、安全
├── 03-Git协作规范.md         # 分支策略、提交规范、Code Review
├── 04-API设计规范.md         # RESTful、错误码、版本管理
├── 05-测试规范.md            # 测试分层、覆盖率、Mock策略
├── 06-文档规范.md            # README、接口文档、变更日志
├── AI协作开发流程.md         # 计划驱动、BDD迭代、双模型协作
├── README.md                 # 项目入口
├── CLAUDE.md                 # 本文件
├── STATUS.md                 # 进度仪表盘
├── dashboard.html            # 可视化仪表盘（自刷新）
├── plan/
│   ├── 背景.md               # 项目背景与设计决策
│   └── 开发清单.md            # 功能点列表与依赖关系
├── docs/
│   ├── research/             # 调研文档
│   ├── specs/                # BDD 功能规格
│   └── decisions/            # 架构决策记录 (ADR)
├── worklogs/                 # 工作日志
└── src/                      # 示例代码（待填充）
```

## 模块依赖图谱

```
README.md  ← 入口
   └── 引用 → 01~06 规范文档 + AI协作开发流程
                  ↓
          plan/开发清单.md  ← 开发计划
                  ↓
          STATUS.md         ← 状态追踪
                  ↓
          dashboard.html    ← 可视化
```

- 7 份规范文档之间无强耦合，可独立阅读
- `AI协作开发流程.md` 定义了本项目的开发方法论
- `CLAUDE.md` 是 Claude 每个新会话的入口文件
