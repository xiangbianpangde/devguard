## 项目概述

一套适用于各类软件项目的通用开发规范模板，覆盖架构、代码、Git、API、测试、文档六个维度，外加 AI 协作开发流程。所有文件为 Markdown 纯文本，Git 管理。

## 模块依赖图谱

本项目为纯文档项目，无代码模块。各文件的引用关系如下：

```
conventions/规范文档 (01-07)     ← 规范正文，彼此独立
        ↕ 互检
docs/specs/BDD规格               ← 验收标准，逐个对应规范文档
        ↓
plan/开发清单.md                  ← 功能点列表，引用 specs/ 中的 BDD
        ↓ 驱动
STATUS.md / dashboard.html       ← 进度追踪，解析 STATUS.md 表格
        ↓ 记录
worklogs/                         ← 每次功能点产出变更日志
```

关键约束：修改规范文档时需同步更新对应的 `docs/specs/` BDD 规格。新增功能点须同时写入 `plan/开发清单.md` 和 `STATUS.md`。

> **新增文件前先查 `meta/FILE_GRAPH.md`**：该文件是文件归类权威（带职责标注的目录树 + 引用关系 + "新文件放哪"决策树）。禁止在根目录堆放散文件。

## 关键决策

| ADR | 决策 | 理由 |
|-----|------|------|
| 0001 | 全部使用 Markdown | Git 友好，AI 可处理，任意编辑器可读写 |
| 0002 | 六维度体系 | 覆盖开发全生命周期，不冗余不遗漏 |
| 0003 | AI 协作流程独立成文 | 方法论与规范内容分离 |
| 0004 | 规范移入 conventions/ 目录 | 参考 AgentHub 组织方式，集中管理 |
| 0005 | 长文件拆文件夹（如 07-ai-workflow） | 单文件 > 300 行即拆，便于定位和阅读 |
