# 豁免清单（Exemption Registry）

> 更新: 2026-07-11
> 本文件是仓库 `[skip-*]` 豁免的唯一登记账；豁免提交必须同步暂存本文件并新增完整使用记录。

## 一、规则

- 只有 §二登记的标记可以使用。
- 每次使用必须在 §三新增一行，写明日期、范围、钩子、Owner 与原因。
- `check_exemption_log.py` 本身没有豁免；登记账读取失败时按 fail-closed 拦截。
- 紧急并不等于免审，豁免记录在下一收束节点复核。

## 二、已登记豁免标记

| 豁免标记 | 适用钩子 | 仅允许场景 |
|---|---|---|
| `[skip-worklog]` | `check_worklog_ref.py` | 收束产物或紧急 hotfix 无独立功能点 worklog |
| `[skip-status]` | `check_status_updated.py` | 仅修订历史 worklog，不改变项目状态 |
| `[skip-worklog-structure]` | `check_worklog_structure.py` | 历史日志迁移，无法补全原始证据 |
| `[skip-file-placement]` | `check_file_placement.py` | 已批准的短期迁移，须在当轮收口 |
| `[skip-updated]` | `check_updated_tag.py` | 纯回滚或保留历史日期的迁移 |
| `[skip-docsync]` | `check_doc_sync.py` | 仅修复历史投影，不新增/删除功能点 |
| `[skip-gate]` | `check_convergence_gate.py` | 建立闸门自身或 Owner 明确批准的紧急修复 |
