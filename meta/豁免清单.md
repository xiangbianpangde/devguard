# 豁免清单（Exemption Registry）

> 更新: 2026-06-11（V2.3 #48：登记 `[skip-html]`）
> 本文件是仓库**豁免使用的唯一登记账**。任何在 commit message 里使用 `[skip-*]` 豁免标记的提交，
> 必须在本次提交中**同时修改本文件**（追加一条「使用记录」），否则 `check_exemption_log.py` 拦截提交。

---

## 一、为什么有这份账

V2.1 起，多个 pre-commit / commit-msg 钩子都提供了 `[skip-*]` 豁免出口（紧急 hotfix / 收束节点等）。
但豁免一旦**散落、零登记**，强制就会名存实亡——谁都能随手 `[skip]` 绕过，且无人追溯。

本账把豁免从"隐式绕过"变成"显式留痕 + 可审计"：

- **每个合法豁免标记**必须在 §二「已登记豁免标记」目录里有定义（未登记的标记 = 拦截）。
- **每次使用豁免**必须在 §三「豁免使用记录」追加一行（提交时强制本文件入暂存区）。
- CI / `--audit` 可汇总历史豁免使用，供收束节点人审计。

> 本账自身**不可被豁免**——`check_exemption_log.py` 是登记器，没有 `[skip-exemption]`。

---

## 二、已登记豁免标记（合法标记目录）

> 新增钩子带新豁免标记时，必须先在此登记，`check_exemption_log.py` 才放行该标记。

| 豁免标记 | 适用钩子 | 含义 / 何时允许 |
|---------|---------|----------------|
| `[skip-worklog]` | check_worklog_ref.py | 本次提交免「引用 worklog」检查——仅收束节点 / 紧急 hotfix |
| `[skip-status]` | check_status_updated.py | 改 worklog 但本次不同步 STATUS.md——仅纯 worklog 修订 |
| `[skip-worklog-structure]` | check_worklog_structure.py | worklog 不含必需段——仅历史 worklog 补登 |
| `[skip-file-placement]` | check_file_placement.py | 新文件暂不符合 FILE_GRAPH——仅过渡期，须当轮收口 |
| `[skip-updated]` | check_updated_tag.py | 受管文件本次不刷新「更新」标签日期——仅纯格式/回滚类提交 |
| `[skip-gate]` | check_convergence_gate.py | 本次提交跳过收束硬闸门——仅建闸门自身 / 已确认即将收束的过渡提交 |
| `[skip-html]` | check_html_artifact.py | HTML 产出物本次免结构契约校验——仅存量产出物大改造过渡期 |

> 待 V2.3 阶段C 补遗落地后追加 `[skip-docsync]`（check_doc_sync.py）。
