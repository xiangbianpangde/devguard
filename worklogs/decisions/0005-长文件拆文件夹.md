# ADR 0005: 长文件拆文件夹

> 日期: 2026-05-26
> 状态: 已采纳

## 背景

07-ai-workflow（AI 协作开发流程）的内容涵盖角色分工、调研、计划、开发、回顾五个阶段，全部放入单文件会导致文件过长（预估 > 500 行），阅读和定位困难。

## 决策

单文件超过 300 行时，拆分为独立目录，每个子主题一个文件。

**理由**：
- 单文件过长时，读者难以通过滚动快速定位具体章节
- 分文件后，每个子主题独立可读，适合按需查阅而非通篇阅读
- Git diff 更聚焦——修改"计划"阶段不会影响"角色分工"的变更历史
- 300 行的阈值与代码编写规范保持一致（`02-coding` 中"单文件 ≤ 500 行"的建议）

**适用条件**：
- 仅当内容确实可以按主题自然拆分时才拆（不强行拆分）
- 拆分后每个子文件应有独立标题和清晰的导航关系
- 目录下需保持编号（01-05）以保证阅读顺序

## 后果

- 07-ai-workflow 从单文件变为包含 5 个子文件的目录
- 其他规范文档（01-06, 08）目前未超过 300 行，保持单文件
- 未来新增规范如需拆分，参照此 ADR 执行
- 文件引用路径需从 `07-ai-workflow_*.md` 更新为 `07-ai-workflow_*/` 目录
