# ADR 0002: 六维度划分

> 日期: 2026-05-26
> 状态: 已采纳

## 背景

需要确定「通用开发规范」覆盖的具体维度及其边界。

## 决策

六个维度及其边界定义：

| 维度 | 文件 | 覆盖范围 | 不覆盖 |
|------|------|---------|--------|
| 架构 | 01 | 分层设计、模块划分、依赖方向 | 具体技术栈选型 |
| 代码 | 02 | 命名、格式、注释、错误处理 | 语言特定语法细节 |
| Git | 03 | 分支策略、Commit 规范、Review 流程 | CI/CD 配置 |
| API | 04 | RESTful 设计、错误码、版本管理 | 具体框架实现 |
| 测试 | 05 | 测试分层、覆盖率、Mock 策略 | 测试框架配置 |
| 文档 | 06 | README、API 文档、变更日志 | 文档工具配置 |

## 理由

- 每个维度「覆盖范围」是跨语言的通用原则，「不覆盖」是平台/语言相关细节
- 边界明确避免规范文档越写越厚
- AI 协作开发流程作为独立文档（07），不属于规范本身，属于方法论

## 后果

- 各规范文档应严格遵循边界，不越界覆盖
- 每条规则需说明适用范围和例外情况
