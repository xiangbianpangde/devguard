# ADR-0007: 可执行治理事实与显式脚手架

> 日期: 2026-07-11
> 状态: 已采纳

## 背景

旧实现用文件存在性和主观百分比声明一致性，递归复制模板也无法保证依赖闭包。CI 还会在检查前修改工作区，使漂移被掩盖。

## 决策

采用三层闭环：跨文件/可执行事实矩阵、隔离 Git 故障注入矩阵、显式 core/optional scaffold manifest。默认门槛分别为 80% 与 90%，任何命令未运行、错误 gate 命中或证据缺失都不计为通过。

## 考虑的替代方案

| 方案 | 优点 | 缺点 |
|---|---|---|
| 文件存在性 + 人工估分 | 实现简单 | 可用空文件制造高分，无法复验 |
| 整目录递归复制 | 操作直观 | 会夹带历史/临时文件，依赖闭包不明确 |
| 完整仓库镜像到每个项目 | 约束最全 | 侵入性和维护成本过高 |
| 可执行事实 + 最小自包含 manifest | 可测、可扩展、部署边界清楚 | 需要维护事实项和 manifest |

## 后果

- 正面：一致性、拦截率和初始化闭包都有可重放证据；Windows/POSIX 使用同一入口。
- 负面：故障注入与全量测试增加 CI 时间；新增治理文件必须同步 manifest、元数据与测试。
- 约束：平台分支保护仍须在 GitHub 真实配置并回读，文档声明不能替代外部状态。
