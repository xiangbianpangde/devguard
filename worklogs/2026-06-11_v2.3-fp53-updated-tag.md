## 关键决策

- **commit-msg 而非 pre-commit 阶段**：因为豁免标记 `[skip-updated]` 在 commit message 里，pre-commit 阶段拿不到 message。对标 `check_status_updated` / `check_file_placement`，用 commit-msg + `git diff --cached` 取 staged 文件。
- **受管文件 = 6 个 markdown 入口/索引**（STATUS/CLAUDE/README/开发清单/FILE_GRAPH/豁免清单）：刻意只收 `> 更新:` 约定的 markdown，不含 `_meta.yaml`（注释式日期，格式不同），保持检测器简单。
- **PostToolUse fail-open**：护栏脚本任何异常都 exit 0 放行，绝不因自身问题卡死会话；只在"受管文件标签非今天"这一确定情形 exit 2 阻断。
- **日期=今天用 `datetime.date.today()`**：钩子是 git/harness 调用的普通脚本（非 workflow 脚本），可用真实日期。
