## 关键决策

- **收束闸门归属 AI 拦截、人执行**：避免"红线却仅靠自觉"。AI 不替人判断该不该收束，但负责在闸门处卡住越界开发——拦截是机械规则，执行仍是人的判断。
- **README 命名保留大写前缀 `README-`**：牺牲"目录自动渲染 README"换取消除同名混淆；用户已知此代价并确认。
- **术语定 `收束`**：仓库 32 文件用收束、仅文件名用收敛，且 dashboard 分类逻辑依赖 `收束` 字样——统一到收束同时修了分类 bug。

## 给下一位的交接

> 新增子目录 README 一律用 `README-<内容>.md`，根目录两文件不动。收束节点是计划预设的硬闸门，AI 到点未收束会拒绝继续——这是预期行为，不是 bug。固定数据（性能/召回率）不要写进规范正文，放调研。
