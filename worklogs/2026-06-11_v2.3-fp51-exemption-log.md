## 做了什么

把散落、零登记的 `[skip-*]` 豁免出口，升级为「显式登记 + 提交时强制 + 可审计」。作为 V2.3 所有硬拦截（#52 收束闸门 / #53 更新时间标签）的前置依赖先行交付。

- [x] 建豁免账 `meta/豁免清单.md`（决策 2：落 `meta/`）——§二 已登记标记目录（现役 4 个）+ §三 使用记录账
- [x] 新增 `scripts/check_exemption_log.py` commit-msg 钩子：commit message 含 `[skip-*]` → 标记须在 §二 登记 + 本次提交须同步暂存 `meta/豁免清单.md`，否则拦截；本钩子自身不可豁免
- [x] 附 `--audit` 模式：扫 git 历史汇总豁免使用 vs 已登记标记（收束节点人审计用）

- [x] `render_meta.py --render pre-commit-config` 投射到根 `.pre-commit-config.yaml`（`--check` pre-commit-config 一致）
- [x] 把 `test_exemption_log` 登记进 `test_05_testing.py` L4 覆盖元测试清单

## 关键决策

- **强制语义 = 「用豁免必须同次提交修改豁免账」**：对标 `check_status_updated`（改 worklog 必带 STATUS）。这样无法在不留痕的情况下使用豁免，是工具可强制的最强语义。
- **合法标记只认 §二 目录表首列**：`registry_markers()` 刻意不扫散文/§三 使用列，避免把散文里提到的 `[skip-exemption]`（明确不存在）或未来标记（`[skip-updated]` 等）误判为已登记。
- **本钩子不可豁免**：它是登记器，没有 `[skip-exemption]`。
- **#51 暂不打 fp 标签**：沿用 #47+ 候选池口径（STATUS↔开发清单 fp 计数保持 46↔46，`check_plan` 不报 WARN）；待 V2.2/V2.3 收束节点统一升级编号 + 重写「46」口径说明。
- **中文路径**：`git diff --cached` 用 `-c core.quotepath=false` 取未转义 UTF-8 路径（`豁免清单.md`）。
