# 工作日志：统一模板规约 + 汇报模板

> 日期: 2026-05-26
> 功能点: #21 统一模板规约 + 汇报模板
> 关联 BDD: docs/specs/10-templates-reporting.md

## 做了什么

补两个缺口：模板缺统一规约、缺汇报模板。

1. **模板索引 + 编写规约** `docs/templates/README.md`：
   - 模板清单（用途 / 何时用 / 实例落点）
   - 统一占位符语法（`{占位}`，禁用 `<占位>`）、文件骨架、命名落点、使用方式
2. **统一 worklog 模板** `docs/templates/worklog模板.md`：
   - 把唯一的格式离群者（`<占位>` + 与在用日志不一致的段落）改为 `{占位}`
   - 定义"必填 + 可选"段，使精简版/完整版都合规
3. **汇报模板** `docs/templates/汇报模板.md`：三档
   - 功能点完成（事件触发）
   - 周期日报/周报（累积素材）
   - 阶段/里程碑（对内进度 + 对上里程碑**合并**一份，按受众取小节）
4. **落点与规格**：新增 `docs/reports/README.md`（汇报落点 + 命名约定）、`docs/specs/10-templates-reporting.md`（BDD 3 场景）。

## 验证结果
- Markdown 链接检查：无死链（含新建文件）
- 占位符审计：`docs/templates/` 下无 `<占位>` 残留
- STATUS.md 解析：21/21 全部识别
- 登记一致性：STATUS / 开发清单 / CLAUDE / FILE_GRAPH 计数与编号同步为 21、specs 00-10

## 关键决策
| 决策 | 原因 | 影响 |
|------|------|------|
| 占位符统一 `{}`、废弃 `<>` | `<>` 与 HTML/泛型混淆 | worklog 模板需改写对齐 |
| 阶段汇报对内/对上合并一份 | 同一事实两个视角，分两份易数字打架 | 一档覆盖两类受众，按小节取用 |
| worklog 定"必填+可选" | 既往精简版与完整版并存 | 两者都合规，不必回改历史日志 |

## 遇到的问题
- worklog 模板与实际在用格式分叉（早前已暴露）：本轮以"必填子集"方式统一，而非强制单一格式。
