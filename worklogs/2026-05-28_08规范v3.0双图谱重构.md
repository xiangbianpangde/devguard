# 08 规范 v3.0 双图谱重构

> 关联 BDD: docs/specs/08-code-understanding.md

## 背景

用户指出：代码图谱应有两种——给 AI 的（机器图谱）和给人的（人类图谱）。当前 08 规范以「图数据库 + Cypher 查询 + Mermaid CODE_MAP」为默认范式，调研数据已证明这条路存在结构性缺陷（跨语言覆盖度 62%、复杂结构 < 40%、构建时间 12 分钟起）。

## 关键决策

**决策**：将 08 规范从单一范式重构为双图谱结构：
- **机器图谱**：CodeGraph（colbymchenry/codegraph，29.8k stars）— tree-sitter → SQLite+FTS5 → MCP 工具，AI 直接查询
- **人类图谱**：Understand-Anything（Lum1104/Understand-Anything，39.8k stars）— tree-sitter+LLM 混合 → JSON → 交互式 Dashboard

**理由**：两个工具都是 v3.0 调研后社区验证的成熟方案，分别服务于 AI（MCP 集成）和人（Web Dashboard）。它们避开了图数据库的全部陷阱——SQLite 零运维、交互式可视化替代手写 Mermaid。
