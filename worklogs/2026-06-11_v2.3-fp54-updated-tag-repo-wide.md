# V2.3 #54 更新标签全仓泛化

> 更新: 2026-06-11 | 功能点 #54（V2.3 阶段B 扩展）| 状态: ✅ 已完成

## 做了什么

用户澄清 #53 原意：**不只 6 个入口文件，所有文档类 .md 都要有「更新」标签、改动后都要刷新**，便于日后文档整理。据此把 #53 的钩子从固定 6 文件泛化到全仓在范围文档，并一次性回填存量。

- [x] **范围泛化**：`check_updated_tag.py` 的固定 `MANAGED` 集合 → `in_scope(rel)` 函数——全仓 git 跟踪 .md，排除 `docs/templates/` / `worklogs/` / `.github/` / `CHANGELOG.md`（用户拍板的排除项）
- [x] **PostToolUse 钩子同步泛化**：`hook_updated_tag_posttooluse.py` 改用 `core.in_scope()`
- [x] **回填工具** `scripts/backfill_updated_tag.py`：为在范围且缺标签的文件插入 `> 更新: <git 末次修改日>`，智能定位（H1 后已有 blockquote 则追加，否则另起）
- [x] **执行回填**：159 个在范围文件中回填 153 个（既有 6 个入口文件跳过）
- [x] **L4 不变量测试**：`test_updated_tag.py` 加「所有在范围 .md 均带标签」+ 范围判定测试
- [x] 同步 `_meta.yaml` 钩子描述 + 豁免清单 §三 登记本次回填豁免 + 四件套 + FILE_GRAPH

## 关键决策

- **回填日期取 git 末次修改日**（用户决策）：真实反映"上次更新"，而非全填今天（那会丢历史、违背"便于整理"初衷）。脚本逐文件 `git log -1 --date=short`。
- **排除 templates/worklogs/.github/CHANGELOG**（用户决策）：模板待填充、worklog 历史冻结、CHANGELOG 自有日期结构、.github 为 PR 模板。
- **回填提交一次性 `[skip-updated]`**：回填把标签设为历史日期，与"提交受管文件须=今天"硬规则天然冲突；这是一次性批量迁移，用豁免并在 `meta/豁免清单.md` §三 登记（dogfood #51）。
- **插入点 markdownlint 安全**：追加进既有 blockquote 块或在 H1 后另起单行 blockquote，前后空行规整；全仓 153 文件 markdownlint **exit 0** 验证无破格。

## 遇到的问题

- 回填会修改 153 个文件且写入历史日期，与 #53 钩子"须=今天"冲突 → 用 `[skip-updated]` 一次性豁免 + 登记，规则本身不放松。
- ruff E501：泛化后注释/docstring 长行 → 缩短修复。
