## 关键决策

- **拦开工的判定锚点 = "同一提交内：staged 提案含待拍板 ∧ 开发清单新翻 ✅"**：两个条件都从 staged diff 机械可算，避免"FP↔提案"映射的脆弱解析；跨提交的遗留待拍板由 `--all` CI 审计可见。
- **agent 工具刻意不含写码/写文件**：交流 agent 的产出是清晰度而非代码，无 Write 工具从机制上防"边讨论边动手"（05-思考设计 反模式）。
- **历史提案追溯登记而非豁免**：让「Owner 决策节」契约全仓统一，CI --all 可常绿；追溯内容只登记事实（已交付=已拍板），不虚构决策过程。
